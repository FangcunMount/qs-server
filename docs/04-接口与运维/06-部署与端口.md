# 部署与端口

## 1. 结论

三个进程独立部署、独立读取配置，但共享明确的网络与依赖拓扑。具体端口以当前环境配置和部署清单为准。

## 2. 依赖

| 进程 | 入站 | 主要出站 |
| --- | --- | --- |
| qs-apiserver | REST、gRPC | MySQL、MongoDB、Redis、MQ、IAM/外部能力 |
| collection-server | REST | apiserver gRPC、Redis/IAM 相关能力 |
| qs-worker | MQ | apiserver internal gRPC、必要信令能力 |

## 3. 发布检查

- migration 与应用版本兼容；
- health/readiness 和依赖探测正确；
- 网络策略只开放必需端口；
- worker 与 relay 的启动顺序不会丢可靠事件；
- 优雅关闭时间覆盖最大在途请求/消息处理窗口。
