# 小程序人格测评前端接入指南

**读者**：微信小程序 / Taro 前端  
**服务**：collection-server（BFF）  
**生产 Base**：`https://collect.fangcunmount.cn/api/v1`  
**OpenAPI**：`GET /api/rest`（源文件 `api/rest/collection.yaml`）  
**关联**：[03-小程序人格测评接入.md](./03-小程序人格测评接入.md)、[12-小程序报告等待接入指南.md](./12-小程序报告等待接入指南.md)

---

## 30 秒结论

| 维度 | 结论 |
| ---- | ---- |
| 路径前缀 | 全部使用 `/api/v1/typology-*`，**禁止** `/personality-*`、`/api/v2/assessments*` |
| 模型目录字段 | `kind` / `sub_kind` / `product_channel` 规范值为 **`typology`**（R128b） |
| 测评/报告字段 | `model.kind` 当前仍为 **`personality`**（evaluation 层未改名） |
| WS 订阅 | `kind` 必须发 **`personality`**，不是 `typology` |
| 主流程 | `typology-assessment-sessions` → `answersheets` → `submit-status` 取 `assessment_id` → `report-status` / WS → `report` |
| ID 类型 | `testee_id`、`assessment_id` 全程用 **字符串**，避免 JS 大整数精度丢失 |

一句话：**URL 用 typology；目录 identity 认 typology；测评/WS 仍认 personality；用 model.code 做产品分支，不要用 algorithm 硬编码。**

## 1. 本次升级（R128b）前端要改什么

### 1.1 必须改

| 项 | 旧写法 | 新写法 |
| -- | ------ | ------ |
| 模型列表/详情 `model.kind` | 只认 `personality` | 以 **`typology`** 为主，兼容读 `personality` |
| `product_channel` | `personality` | **`typology`**（兼容旧值） |
| API 路径 | `/personality-*` | `/typology-*` |

### 1.2 不要改

| 项 | 说明 |
| -- | ---- |
| `GET .../report` 返回的 `model.kind` | 仍为 `"personality"`，collection 服务端据此校验人格线 |
| WS `subscribe.kind` | 仍为 `"personality"` |
| `payload_format` | 字符串仍为 `assessmentmodel.personality.typology.v1`（机制别名，非 API kind） |
| 产品分支逻辑 | 用 **`model.code`**（如 `MBTI_OEJTS`），不要用 `algorithm === 'mbti'` 写死 |

### 1.3 推荐归一化函数

```typescript
/** 模型目录层：typology / personality 均视为人格测评 */
export function isTypologyCatalogModel(model: { kind?: string }): boolean {
  const k = model.kind ?? '';
  return k === 'typology' || k === 'personality';
}

/** 测评/报告层：当前服务端仍输出 personality */
export function isTypologyAssessmentModel(model: { kind?: string }): boolean {
  const k = model.kind ?? '';
  return k === 'personality' || k === 'typology';
}
```

## 2. 鉴权与通用约定

### 2.1 请求头

| Header | 必填 | 说明 |
| ------ | ---- | ---- |
| `Authorization` | 业务接口必填 | `Bearer <JWT>` |
| `Content-Type` | POST 必填 | `application/json` |
| `X-Request-Id` | 提交答卷建议带 | UUID，幂等 + `submit-status` 查询键 |

### 2.2 响应信封

```json
{ "code": 0, "message": "ok", "data": { } }
```

`code !== 0` 为业务错误；HTTP 状态码与 `code` 可能不一致（提交答卷为 **202**）。

### 2.3 匿名 vs 登录

| 接口 | 登录 |
| ---- | ---- |
| `GET /typology-models`、`GET /typology-models/categories` | 可匿名 |
| `GET /typology-models/{code}` | **需 JWT** |
| session / 答卷 / 测评 / 报告 / testee | **需 JWT** |

## 3. 字段字典（人格测评）

### 3.1 模型目录（session / typology-models）

| 字段 | 典型值 | 说明 |
| ---- | ------ | ---- |
| `kind` | `typology` | 模型族（R128b canonical） |
| `sub_kind` | `typology` | 子机制 |
| `product_channel` | `typology` | 产品通道 |
| `algorithm` | `mbti` / `sbti` / `bigfive` / `personality_typology` | 算法实现，**不做产品分支** |
| `algorithm_family` | `factor_classification` | 固定推导 |
| `payload_format` | `assessmentmodel.personality.typology.v1` | 载荷格式（字符串含 personality 为历史命名） |
| `decision_kind` | `pole_composition` 等 | 决策机制 |
| `code` | `MBTI_OEJTS` | **产品主键，前端路由用这个** |

### 3.2 测评/报告（typology-assessments）

| 字段 | 典型值 | 说明 |
| ---- | ------ | ---- |
| `model.kind` | `personality` | evaluation 层过滤值（与目录 `typology` 并存） |
| `model.sub_kind` | `typology` | |
| `model.algorithm` | `mbti` | |
| `model.product_channel` | `typology` | |

## 4. 推荐全链路（Happy Path）

```text
1. POST /typology-assessment-sessions     → 拿题 + submit_contract
2. POST /answersheets                     → 202 + request_id
3. GET  /answersheets/submit-status       → answersheet_id；测评落库后附带 assessment_id
4. （兜底）GET /typology-assessments?testee_id= → 按 answer_sheet_id 匹配 assessment_id
5. GET  /typology-assessments/{id}/report-status  → 短轮询（或 WS）
6. GET  /typology-assessments/{id}/report         → 完整报告
```

> **R121**：`GET /answersheets/{id}/assessment` 已下线，勿调用。`session.endpoints.assessment_by_answer_sheet` 仅为历史模板。

## 5. 接口明细

### 5.1 开始测评会话（推荐入口）

`POST /api/v1/typology-assessment-sessions`

**请求**

```json
{
  "model_code": "MBTI_OEJTS",
  "testee_id": "618855887087350318"
}
```

`testee_id` 支持字符串或数字；**建议字符串**。

**响应 `data`**

```json
{
  "model": {
    "code": "MBTI_OEJTS",
    "version": "1.0.0",
    "title": "MBTI 人格类型测评",
    "algorithm": "mbti",
    "questionnaire_code": "MBTI_OEJTS",
    "questionnaire_version": "1.0.0",
    "status": "published",
    "question_count": 32,
    "kind": "typology",
    "sub_kind": "typology",
    "product_channel": "typology",
    "algorithm_family": "factor_classification",
    "payload_format": "assessmentmodel.personality.typology.v1",
    "decision_kind": "pole_composition"
  },
  "questionnaire": {
    "code": "MBTI_OEJTS",
    "version": "1.0.0",
    "title": "...",
    "questions": [
      {
        "code": "Q_EI",
        "type": "Radio",
        "title": "...",
        "options": [{ "code": "A", "content": "...", "score": 1 }]
      }
    ]
  },
  "submit_contract": {
    "questionnaire_code": "MBTI_OEJTS",
    "questionnaire_version": "1.0.0",
    "testee_id": "618855887087350318"
  },
  "endpoints": {
    "submit_answer_sheet": "/api/v1/answersheets",
    "assessment_by_answer_sheet": "/api/v1/answersheets/{answersheet_id}/assessment",
    "wait_report": "/api/v1/typology-assessments/{assessment_id}/wait-report?testee_id=618855887087350318",
    "report": "/api/v1/typology-assessments/{assessment_id}/report?testee_id=618855887087350318"
  }
}
```

**注意**

- session **不创建** assessment，响应里**没有** `assessment_id`。
- 必须使用返回的 **`questionnaire_version`** 提交，不能用「问卷最新版」。

### 5.2 提交答卷

`POST /api/v1/answersheets`

**请求**

```json
{
  "questionnaire_code": "MBTI_OEJTS",
  "questionnaire_version": "1.0.0",
  "testee_id": "618855887087350318",
  "idempotency_key": "mp-submit-20260708-001",
  "answers": [
    {
      "question_code": "Q_EI",
      "question_type": "Radio",
      "score": 1,
      "value": "{\"option\":\"A\"}"
    }
  ]
}
```

| 字段 | 说明 |
| ---- | ---- |
| `question_type` | 与题目 `type` 一致，如 `Radio` |
| `score` | 取所选 option 的 `score` |
| `value` | **JSON 字符串**，如 `{"option":"A"}` |

**响应 HTTP 202**

```json
{
  "code": 0,
  "message": "accepted",
  "data": {
    "status": "queued",
    "request_id": "<uuid>"
  }
}
```

---

### 5.3 查询提交状态

`GET /api/v1/answersheets/submit-status?request_id={request_id}`

```json
{
  "status": "done",
  "answersheet_id": "9001",
  "assessment_id": "8001",
  "updated_at": 1719750000
}
```

`status` 非终态时继续轮询。`status=done` 后若尚无 `assessment_id`，说明异步测评尚未落库，**继续轮询同一接口**（人格/医学通用）；出现 `assessment_id` 后进入报告等待步骤。

---

### 5.4 获取 assessment_id

**推荐（人格/医学通用）**：继续轮询 `GET /api/v1/answersheets/submit-status?request_id={request_id}`，在 `status=done` 且测评落库后响应附带 `assessment_id`。

**人格兜底**：若长时间无 `assessment_id`，可 `GET /api/v1/typology-assessments?testee_id={testee_id}&page=1&page_size=20`，按 `answer_sheet_id` 匹配。

列表项含 `answer_sheet_id`（与 `answersheet_id` 对应）：

```json
{
  "items": [
    {
      "id": "8001",
      "answer_sheet_id": "9001",
      "questionnaire_code": "MBTI_OEJTS",
      "status": "submitted",
      "model": { "kind": "personality", "code": "MBTI_OEJTS", "algorithm": "mbti" }
    }
  ],
  "total": 1,
  "page": 1,
  "page_size": 20
}
```

**兜底逻辑**：列表匹配 `answer_sheet_id === answersheet_id`，取 `id` 作为 `assessment_id`。

```typescript
async function waitAssessmentId(
  testeeId: string,
  answerSheetId: string,
  opts = { intervalMs: 2000, maxAttempts: 30 }
): Promise<string> {
  for (let i = 0; i < opts.maxAttempts; i++) {
    const res = await api.get('/typology-assessments', {
      params: { testee_id: testeeId, page: 1, page_size: 20 },
    });
    const hit = res.data.items.find((x) => x.answer_sheet_id === answerSheetId);
    if (hit?.id) return hit.id;
    await sleep(opts.intervalMs);
  }
  throw new Error('assessment_id timeout');
}
```

### 5.6 报告等待

详见 [12-小程序报告等待接入指南.md](./12-小程序报告等待接入指南.md)。

**推荐**：`report-status` 短轮询；WS 为增强（生产默认关闭）。

`GET /api/v1/typology-assessments/{id}/report-status?testee_id={testee_id}`

```json
{
  "status": "processing",
  "stage": "queued",
  "message": "报告排队生成中",
  "next_poll_after_ms": 3000,
  "updated_at": 1719750000
}
```

终态：`interpreted` | `failed` | `completed`（`completed` 等同 `interpreted`）。

**WebSocket**（`report_events.enabled=true` 时）：

```text
WSS /api/v1/report-events
Header: Authorization: Bearer <token>
```

订阅帧：

```json
{"op":"subscribe","assessment_id":"8001","kind":"personality","testee_id":"618855887087350318"}
```

`GET /api/v1/typology-assessments/{id}/report?testee_id={testee_id}`

**仅在 `report-status` 终态为 `interpreted` 后请求**，否则 404。

## 8. 前端状态机

```text
idle
  → session_ok（有题目 + submit_contract）
  → submitting（POST answersheets 202）
  → submit_polling（submit-status）
  → submit_done（有 answersheet_id）
  → assessment_polling（列表匹配 answer_sheet_id）
  → assessment_ready（有 assessment_id）
  → report_waiting（report-status 或 WS）
  → report_ready（GET report）
```

**本地持久化建议**：`model_code`、`testee_id`、`request_id`、`answersheet_id`、`assessment_id`。

## 9. 错误处理速查

| 场景 | HTTP / 行为 | 处理 |
| ---- | ----------- | ---- |
| session 404 | model 未发布或 code 错误 | 检查 `model_code` |
| submit 429 | 限流 | 退避重试，保留同一 `X-Request-Id` |
| report 404 + status 非终态 | 报告未生成 | 继续 report-status，勿打 report |
| report 404 + testee 错误 | 越权防护 | 检查 `testee_id` 与测评归属一致 |
| WS 404 | 生产未开 report_events | 仅用 report-status |
| `model.kind` 判断失败 | 只认单一值 | 用本文 §1.3 归一化函数 |
