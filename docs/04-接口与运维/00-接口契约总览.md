# 接口契约总览

## 1. 结论

REST 以 OpenAPI 为准，gRPC 以 proto 为准，事件/信令以 YAML 契约为准。本文只给路由，不复制 schema。

## 2. 契约地图

| 类型 | 事实源 | 消费方 |
| --- | --- | --- |
| apiserver REST | `api/rest/apiserver.yaml` | 管理后台、运维、公开入口 |
| collection REST | `api/rest/collection.yaml` | 小程序/前台 |
| gRPC | `api/grpc/proto` | collection、worker、内部服务 |
| events | `configs/events.yaml` | outbox relay、MQ、worker |
| signals | `configs/signals.yaml` | 在线 waiter、缓存失效/预热 |

## 3. 更新规则

- handler/DTO 变化时重新生成 REST 契约并执行 `make docs-verify`；
- proto 变化时重新生成代码并运行调用方测试；
- 事件/信令变化时更新 catalog、注册、配置和契约测试；
- prose 文档不得独立发明字段、默认值或鉴权规则。
