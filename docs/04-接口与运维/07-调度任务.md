# 调度任务

## 1. 结论

调度任务必须从当前配置和 container runner 反向核对，不能根据旧文档假设已启用。

## 2. 任务类别

- Outbox relay 与失败重试；
- Plan 任务打开、过期、通知与对账；
- Statistics 夜间 publish 与人工 validate/repair/resume-cache；
- cache 预热、热集与治理；
- resilience 操作审计恢复；
- 其它平台维护任务。

## 3. 运维要求

每个任务应有启用开关、周期/批量、互斥或租约、幂等、失败重试、运行账本或进度、指标和安全停止方式。Statistics 不使用持久扫描 checkpoint，而以日期修复窗口、稳定 FactKey 和 SyncRun 实现重跑与恢复。

## 4. 核对方法

从配置字段找到 runner 装配，再找到 application service 和持久化状态；最后确认 health/metrics/logging 能观察到它实际运行。

Statistics 当前只保留 `statistics_sync`：上海时间 00:30 按机构串行调用唯一 Coordinator 执行 `publish`。Behavior Journey Scan、Pending Reconcile 和 V1 三段同步均已退役，不应再出现在配置或运行清单中。
