# 调度任务

## 1. 结论

调度任务必须从当前配置和 container runner 反向核对，不能根据旧文档假设已启用。

## 2. 任务类别

- Outbox relay 与失败重试；
- Plan 任务打开、过期、通知与对账；
- Statistics 同步、checkpoint、pending reconcile；
- cache 预热、热集与治理；
- resilience 操作审计恢复；
- 其它平台维护任务。

## 3. 运维要求

每个任务应有启用开关、周期/批量、互斥或租约、幂等、失败重试、进度/checkpoint、指标和安全停止方式。

## 4. 核对方法

从配置字段找到 runner 装配，再找到 application service 和持久化状态；最后确认 health/metrics/logging 能观察到它实际运行。
