# 常见排障

**本文回答**：接口与运维层常见问题如何从现象定位到 REST、gRPC、配置、端口、调度、健康检查、Redis/Resilience/Event/Security 等模块；每类问题先看什么、再查什么。

---

## 1. REST 404

检查顺序：

1. 是 apiserver 还是 collection。
2. 路径前缀是否正确：`/api/v1`、`/internal/v1`、`/api/rest`。
3. route 是否注册。
4. OpenAPI 是否滞后。
5. 是否访问了历史旧路径。
6. 是否 host port 指错进程。

常见错误：

- 把 `/api/rest` 当业务 API。
- 用 collection 调后台发布接口。
- 用 apiserver 调前台 submit-status。

---

## 2. REST 401 / 403

401 检查：

1. Authorization header。
2. IAM enabled。
3. TokenVerifier。
4. JWT 是否过期。
5. ForceRemoteVerification。

403 检查：

1. tenant_id。
2. numeric org scope。
3. active operator。
4. AuthzSnapshot。
5. CapabilityDecision。
6. IAM permissions。

---

## 3. Submit 返回 429

先区分来源：

| 来源 | 现象 |
| ---- | ---- |
| RateLimit | `rate_limited` outcome |
| SubmitQueue full | `queue_full` outcome |
| SubmitGuard contention | submit already in progress / ResourceExhausted |

排查：

1. collection rate limit。
2. queue depth/capacity。
3. worker count。
4. apiserver gRPC。
5. Mongo durable submit。
6. SubmitGuard lock。

---

## 4. gRPC Unimplemented

检查：

1. proto 方法是否存在。
2. apiserver Registry 是否注册对应 service。
3. Container deps 是否 nil。
4. 是否连到正确 apiserver 版本。
5. client stub 是否和 proto 同步。

---

## 5. gRPC Unauthenticated / PermissionDenied

Unauthenticated：

- token metadata。
- service auth。
- TokenVerifier。
- mTLS。
- Auth enabled。

PermissionDenied：

- mTLS identity match。
- ACL。
- AuthzSnapshot。
- capability/resource action。

---

## 6. 端口不通

检查：

1. 配置监听地址。
2. 容器是否启动。
3. 容器内端口。
4. 宿主机映射。
5. HTTP/HTTPS 协议。
6. 防火墙/安全组。
7. gRPC 是否只在容器网络开放。

---

## 7. 调度任务不跑

检查：

1. apiserver scheduler config。
2. scheduler manager 启动日志。
3. Redis leader lock。
4. 多实例 contention。
5. 业务 service 错误。
6. 是否误以为 crontab 仍是默认路径。

---

## 8. worker 不消费或积压

检查：

1. MQ provider。
2. topic/channel。
3. worker concurrency。
4. handler registry。
5. duplicate suppression。
6. internal gRPC。
7. Ack/Nack。
8. poison message。

---

## 9. Redis degraded

检查：

1. `/governance/redis`。
2. `qs_cache_family_available`。
3. family/profile/namespace。
4. Redis password/network。
5. fallback default。
6. last_error。

---

## 10. Healthz ok 但业务失败

说明 HTTP router 活着，但依赖可能 degraded。

继续查：

- readyz。
- governance redis。
- governance resilience。
- DB/Mongo logs。
- IAM。
- event/outbox。

---

## 11. CPU / 内存异常

CPU：

1. pprof profile。
2. top routes。
3. hot goroutines。
4. JSON/marshal。
5. Mongo/DB loops。

内存：

1. pprof heap。
2. queue depth。
3. cache payload。
4. goroutine count。
5. large response。

---

## 12. 外部通知失败

检查：

1. notification config。
2. WeChatAppID / AppID / AppSecret。
3. recipient resolver。
4. templateID。
5. template keys。
6. WeChat SubscribeSender 是否真实发送还是 seam。
7. OSS object store，如涉及 QR。

---
