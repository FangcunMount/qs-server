# 健康检查与观测

## 1. 结论

存活、依赖就绪、服务准入和业务可用性是不同层次。单个 `/health` 返回成功不证明异步链、数据库或报告生成正常。

## 2. 检查层次

| 层次 | 关注点 |
| --- | --- |
| liveness | 进程是否存活、事件循环是否卡死 |
| readiness | 必需依赖与 transport 是否可服务 |
| serving readiness | 已完成必要初始化，是否可在依赖降级时继续承接受保护的低流量 |
| dependency | MySQL/Mongo/Redis/MQ/gRPC/IAM 状态 |
| business | 提交、outbox lag、worker 消费、Outcome、Report、统计投影 |
| governance | queue/backpressure/lease/cache/resilience 状态与审计 |

## 3. 排障关联键

优先使用 request ID、event ID、AnswerSheet/Assessment/Report 标识、租户/组织标识串联日志与指标；不要记录 token、答案或报告正文。
