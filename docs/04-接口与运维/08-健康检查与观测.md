# 健康检查与观测

**本文回答**：qs-server 的健康检查、metrics、pprof、governance/status 入口分别在哪里；`/healthz`、`/health`、`/readyz` 的语义如何区分；观测排障时如何从接口层进入 Redis/Resilience/Event/Security 子系统。

---

## 30 秒结论

| 入口 | 说明 |
| ---- | ---- |
| `/healthz` | GenericAPIServer 通用健康检查，返回 status ok |
| `/health` | 业务/进程 health handler |
| `/readyz` | 业务/依赖 ready handler |
| `/metrics` | gin-prometheus 与自定义 Prometheus 指标 |
| `/debug/pprof/*` | pprof 性能剖析 |
| `/governance/redis` | Redis family/runtime 只读状态 |
| `/governance/resilience` | Resilience 只读状态 |
| `/internal/v1/*/status` | apiserver internal 状态入口 |

一句话概括：

> **healthz 说明进程/HTTP router 活着；governance/status 解释依赖和横切能力是否 degraded。**

---

## 1. GenericAPIServer 通用入口

GenericAPIServer 安装：

- `/healthz`。
- `/metrics`。
- `/debug/pprof`。
- `/version`。

前提由 config 控制：

- Healthz。
- EnableMetrics。
- EnableProfiling。

Run 阶段会自 ping `/healthz`，10 秒内不成功则返回启动错误。

---

## 2. 业务健康入口

apiserver 和 collection 额外注册：

- `/health`。
- `/readyz`。
- `/ping`。
- `/governance/redis`。
- `/governance/resilience`，collection 有。

这些入口由各进程 health handler 提供。

---

## 3. Metrics

当前重要指标：

- `qs_cache_*`：cache/family/warmup/lock。
- `qs_query_cache_version_total`。
- `qs_runtime_component_ready`。
- `qs_resilience_*`。
- gin HTTP metrics。

详细规范见：

- `03-基础设施/observability/01-Metrics指标体系.md`。

---

## 4. Pprof

默认 EnableProfiling=true 时：

```text
/debug/pprof
```

用于：

- CPU high。
- heap growth。
- goroutine leak。
- blocking。
- mutex contention。

生产访问必须受控，不应公网开放。

---

## 5. Governance 状态入口

| Endpoint | 说明 |
| -------- | ---- |
| `/governance/redis` | 当前进程 Redis runtime/family 状态 |
| `/governance/resilience` | 当前进程 resilience snapshot |
| `/internal/v1/cache/governance/status` | apiserver cache governance 状态 |
| `/internal/v1/cache/governance/hotset` | hotset top |
| `/internal/v1/events/status` | event catalog/outbox 只读状态 |
| `/internal/v1/resilience/status` | apiserver resilience 只读状态 |

默认只读。manual warmup / repair complete 属于受控 action。

---

## 6. 常见观测路径

| 现象 | 先看 |
| ---- | ---- |
| 429 | resilience decision metrics |
| queue full | queue depth/status |
| Redis degraded | family status / `qs_cache_family_available` |
| report 慢 | worker event + pprof + DB latency |
| permission denied | AuthzSnapshot / capability logs |
| gRPC timeout | grpc logs + backpressure + pprof |
| CPU high | pprof profile |
| memory high | pprof heap |

---

## 8. 边界

- `/healthz` 不等于完整 readiness。
- `/metrics` 不应暴露公网。
- `/debug/pprof` 必须受控。
- governance endpoint 不应随意执行修复。
- 高基数 ID 不应放 metrics label。
