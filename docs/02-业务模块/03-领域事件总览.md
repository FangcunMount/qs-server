# 领域事件总览

**本文回答**：当前业务模块通过哪些事件协作，哪些事件是可靠出站，哪些事件只做尽力投递或读侧投影。

事件契约以 [`configs/events.yaml`](../../configs/events.yaml) 为准。本文只解释业务含义和模块边界。

---

## 1. 事件分组

| Topic | 事件组 | 主要模块 | 语义 |
| ----- | ------ | -------- | ---- |
| `qs.survey.lifecycle` | `questionnaire-lifecycle` | `survey`、`assessment-model` | 问卷和量表生命周期变化 |
| `qs.evaluation.lifecycle` | `assessment-lifecycle` | `survey`、`evaluation`、`interpretation-model` | 答卷、测评和报告生命周期 |
| `qs.analytics.behavior` | `analytics-behavior` | `statistics` | 行为足迹和服务过程投影 |
| `qs.plan.task` | `task-lifecycle` | `plan` | 任务开放、完成、过期、取消 |

---

## 2. 主链路事件

| 事件 | 产生模块 | 消费方向 | Delivery | 说明 |
| ---- | -------- | -------- | -------- | ---- |
| `questionnaire.changed` | `survey` | 缓存、目录、读侧刷新 | `best_effort` | 问卷生命周期变化 |
| `scale.changed` | `assessment-model` | 目录、缓存、模型资产刷新 | `best_effort` | 历史事件名保留，语义归入模型资产层 |
| `answersheet.submitted` | `survey` | `evaluation` | `durable_outbox` | 答卷提交后的可靠执行入口 |
| `assessment.submitted` | `evaluation` | Worker / 执行处理 | `durable_outbox` | 测评进入可执行状态 |
| `assessment.interpreted` | `evaluation` | 报告与投影 | `durable_outbox` | 测评已完成解读或 outcome 投影 |
| `assessment.failed` | `evaluation` | 补偿、告警、统计 | `durable_outbox` | 测评失败事实 |
| `report.generated` | `interpretation-model` | 查询、重点关注同步、统计 | `durable_outbox` | 最终报告生成事实 |

---

## 3. 行为投影事件

| 事件 | 统计含义 |
| ---- | -------- |
| `footprint.entry_opened` | 用户打开测评入口 |
| `footprint.intake_confirmed` | 用户完成一次接入 |
| `footprint.testee_profile_created` | 新建受试者档案 |
| `footprint.care_relationship_established` | 建立服务或看护关系 |
| `footprint.care_relationship_transferred` | 转移服务或看护关系 |
| `footprint.answersheet_submitted` | 提交答卷 |
| `footprint.assessment_created` | 形成测评 |
| `footprint.report_generated` | 产出报告 |

这些事件服务 `statistics` 读侧投影，不应反向驱动核心写模型。

---

## 4. 任务生命周期事件

| 事件 | 产生模块 | Delivery | 说明 |
| ---- | -------- | -------- | ---- |
| `task.opened` | `plan` | `best_effort` | 任务入口已开放 |
| `task.completed` | `plan` | `best_effort` | 任务已完成 |
| `task.expired` | `plan` | `best_effort` | 任务已过期 |
| `task.canceled` | `plan` | `best_effort` | 任务已取消 |

任务事件说明计划层状态，不代表测评已经计分或报告已经生成。

---

## 5. 事件边界规则

1. `survey` 发布作答事实，不调用执行细节。
2. `evaluation` 发布执行事实，不依赖统计结果。
3. `interpretation-model` 发布报告事实，不修改统计读模型。
4. `statistics` 消费事件和读模型，不反向改变主业务状态。
5. 新增事件必须先更新 `configs/events.yaml`，再更新本文和相关模块文档。
