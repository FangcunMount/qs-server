# 术语表

**本文回答**：业务模块文档中常见概念如何定义，以及哪些概念不能混用。

---

## 1. 核心术语

| 术语 | 定义 | 所属模块 |
| ---- | ---- | -------- |
| `Questionnaire` | 一份问卷定义，包含题目结构、展示规则、提交规格和生命周期 | `survey` |
| `Question` | 问卷中的题目，描述题型、选项、必填和校验规则 | `survey` |
| `AnswerSheet` | 一次提交后的答卷事实，记录谁在什么上下文下答了什么 | `survey` |
| `AnswerItem` | 答卷中的单题答案 | `survey` |
| `AssessmentModel` | 可发布、可版本化、可被执行层消费的测评模型资产 | `model-catalog` |
| `AssessmentKind` | 测评模型分类，例如医学量表、人格模型等 | `model-catalog` |
| `AssessmentModelSnapshot` | 发布后冻结的模型快照，供执行和追溯使用 | `model-catalog` |
| `ModelBinding` | 模型与问卷、入口或解释模型之间的绑定关系 | `model-catalog` |
| `Assessment` | 一次具体测评行为或执行实例 | `evaluation` |
| `EvaluationRun` | 一次执行尝试，承载运行状态、错误和重试语义 | `evaluation` |
| `EvaluationResult` | 测评执行生成的结构化结果 | `evaluation` |
| `InterpretationModel` | 把结构化结果解释成报告内容的规则资产 | `interpretation` |
| `InterpretReport` | 面向用户或业务查询的最终解释报告实例 | `interpretation` |
| `Actor` | 测评业务中的参与者视图 | `actor` |
| `Testee` | 被测评的人，不等同于 IAM 用户 | `actor` |
| `AssessmentPlan` | 测评计划，描述谁、何时、按什么周期测 | `plan` |
| `AssessmentTask` | 计划拆出的应测任务实例 | `plan` |
| `StatisticProjection` | 由事件或读模型聚合出的统计投影 | `statistics` |
| `ProductChannel` | 产品/运营视角的测评分类（可写、持久化），如 `medical_scale`、`behavior_ability` | `model-catalog` |
| `AlgorithmFamily` | 执行算法族（派生、只读）。枚举：`factor_scoring` 等；包名：`scoring` 等 — 见 [机制迁移对照表](./mechanism-oriented-migration.md#包名与-algorithmfamily-对照表) | `model-catalog` |
| 机制包名 | 执行/报告代码组织单元：`scoring`、`typology`、`norming`、`task_performance`；与 `AlgorithmFamily` 枚举一一对应（`task_performance` 两端同名） | `evaluation` / `interpretation` |
| `Factor` / `FactorSnapshot` | ModelCatalog 发布快照中的通用维度构件：题目归属、计分策略、解释/常模引用 | `model-catalog` |
| `Dimension` / `DimensionResult` | Evaluation 输出侧的维度结果；对应 catalog 侧 Factor 的执行产物 | `evaluation` |

## 2. 必须区分的反例

| 不是 | 正确理解 |
| ---- | -------- |
| `AnswerSheet` 不是 `AssessmentResult` | 答卷记录“答了什么”，测评结果记录“算出了什么” |
| `AssessmentModel` 不是 `Questionnaire` | 模型是规则资产，问卷是采集结构 |
| `InterpretReport` 不是 `Evaluation` | 报告是解释结果，Evaluation 是执行过程和结构化结果 |
| `PlanTask` 不是 `AnswerSheet` | 任务是应测安排，答卷是用户提交事实 |
| `Testee` 不是 IAM User | Testee 是测评业务身份，IAM User 是认证主体 |
| `Statistics` 不是事实源 | Statistics 是读侧投影，主事实归属对应业务模块 |
| `scale` 不是独立核心模块 | Scale 是 `model-catalog` 下的医学量表模型资产或兼容注册名 |
| `interpretation` 不是文档主名称 | 代码包仍叫 `interpretation`，文档业务语言是 `interpretation` |
| `Factor` 不是 `scale` 私有 | Factor 是 ModelCatalog 通用构件；scale 是早期医学量表主链路 |
| `Factor` 不是 `DimensionResult` | Factor 在快照里描述规则；Dimension 是一次执行的结构化得分 |
| `ProductChannel` 不是 `Kind` | 前者是产品分类；后者是模型族/兼容执行入口 |
| `AlgorithmFamily` 不是 `Algorithm` | 前者是执行族抽象（枚举 `factor_*`）；后者是 brief2/mbti/spm 等具体算法 |
| `AlgorithmFamily` 不是机制包名 | 枚举如 `factor_scoring`；包名如 `scoring` — 见机制迁移对照表 |

## 3. 命名约定

| 文档写法 | 代码写法 | 说明 |
| -------- | -------- | ---- |
| `model-catalog` | `modelcatalog` | 文档用业务名称，代码包保持现状 |
| `interpretation` | `interpretation` | 文档表达解释模型 / 报告产出层 |
| `statistics` | `statistics` | 文档目录和代码包都采用复数 |
| `scale` | `modelcatalog` legacy register name | 只作为兼容或具体模型资产出现 |
| `personalitymodel` | `modelcatalog` legacy register name | 只作为兼容或具体模型资产出现 |
