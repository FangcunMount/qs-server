# 核心业务全链路

## 1. 结论

```text
创建/发布 Questionnaire
  + 创建/编辑/发布 Assessment Model
  + 绑定 Questionnaire 与 Published Model
  -> 提交 AnswerSheet
  -> 创建并执行 Evaluation
  -> 提交 Outcome
  -> 生成 InterpretReport
  -> 更新报告查询与统计投影
```

## 2. 事务与异步边界

- 答卷事实与 `answersheet.submitted` 的可靠出站意图同事务提交。
- Evaluation 请求、Outcome、失败分别通过明确状态和可靠事件推进。
- 报告生成读取冻结 Outcome；生成成功/失败有独立事件。
- worker 只驱动 application service，不能跨过 apiserver 事务边界。
