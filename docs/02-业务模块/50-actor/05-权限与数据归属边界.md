# 权限与数据归属边界

## 1. 业务目标

区分 IAM 认证授权和测评业务数据归属，避免把登录权限误当作所有业务数据权限。

---

## 2. 边界表

| 问题 | IAM | Actor |
| ---- | --- | ----- |
| 你是谁 | 认证主体、token、组织身份 | 业务参与者、受试者、操作者 |
| 能登录吗 | 负责 | 不负责 |
| 能代表谁测评 | 不直接负责 | 负责关系和场景判断 |
| 报告属于谁 | 不直接负责 | 提供受试者和组织上下文 |
| 数据统计按谁聚合 | 不直接负责 | 提供人员、组织维度 |

---

## 3. 关键规则

- 登录成功不代表可访问任意 Testee。
- 操作者角色需要结合组织和业务关系判断。
- 报告和统计查询必须带业务归属上下文。

---

## 4. 反例

- 不要把 IAM User ID 直接写成 Testee ID。
- 不要用前端入口参数替代服务端关系校验。
- 不要让 Statistics 反向决定 Actor 权限。
