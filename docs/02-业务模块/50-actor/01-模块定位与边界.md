# Actor 模块定位与边界

## 1. 模块定位

Actor 是业务参与者上下文层。它把 IAM 的认证主体转换成测评业务可以理解的受试者、从业者、操作者和关系上下文。

---

## 2. 负责范围

| 范围 | 说明 |
| ---- | ---- |
| 受试者 | `Testee` 档案、标签、重点关注 |
| 操作者 | `Operator`、`Clinician` 视图 |
| 业务关系 | 服务关系、照护关系、归属关系 |
| 入口上下文 | Assessment entry 和访问上下文 |

---

## 3. 不负责范围

| 不负责 | 归属 |
| ------ | ---- |
| 登录认证 | IAM |
| 问卷定义 | `survey` |
| 测评执行 | `evaluation` |
| 报告生成 | `interpretation` |

---

## 4. 边界反例

- `Testee` 不是 IAM User。
- `Operator` 不是完整 IAM 权限模型。
- `ActorContext` 不是 `AnswerSheet`。
- 业务关系不是统计指标。
