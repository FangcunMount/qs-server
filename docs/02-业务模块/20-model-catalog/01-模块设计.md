# ModelCatalog 模块设计

**本文回答**：ModelCatalog 模块负责什么、当前代码如何分层、上下游是谁，以及哪些边界仍处于过渡状态。

---

## 1. 模块定位

ModelCatalog 是测评模型资产目录。

它管理后台可配置的测评模型资产，并在发布时生成运行侧可读取的发布快照。

一句话：

```text
ModelCatalog 管“如何解释答卷的模型定义”，不管“某次答卷解释出了什么结果”。
```

核心映射：

```text
questionnaire_code + questionnaire_version -> published model snapshot
```

当前代码里发布快照的 port 名称仍是 `PublishedModel`；目标设计里称为 `AssessmentSnapshot`。

---

## 2. 当前代码分层

| 层 | 当前代码 | 职责 |
| -- | -------- | ---- |
| Domain | `internal/apiserver/domain/modelcatalog` | 模型身份、产品分类、`AssessmentModel`、Factor、payload schema、legacy scale aggregate |
| Application | `internal/apiserver/application/modelcatalog` | 后台模型创建、编辑、绑定、发布、查询、选项和能力矩阵 |
| Port | `internal/apiserver/port/modelcatalog` | 草稿仓储、发布读模型、运行读取、问卷解析接口 |
| Infra Mongo | `internal/apiserver/infra/mongo/modelcatalog` | `assessment_models`、`published_assessment_models` 持久化 |
| Runtime bridge | `internal/apiserver/infra/modelcatalog`、`internal/apiserver/infra/ruleset`、`internal/apiserver/infra/evaluationinput` | 发布模型读取、payload decode、evaluation input 适配 |
| Legacy scale | `internal/apiserver/domain/modelcatalog/scoring/definition`、`internal/apiserver/infra/mongo/scale` | 旧医学量表 authoring 和 `scales` collection |

当前最重要的事实是：**ModelCatalog 已经有统一草稿聚合和发布读模型，但 domain 包内部仍混合了目标模型、执行 payload 和 legacy 聚合。**

---

## 3. 模块负责什么

| 职责 | 当前实现 |
| ---- | -------- |
| 管理草稿模型配置 | `port.ModelRepository` + `assessment_models` |
| 管理产品分类和算法身份 | `ProductChannel`、`Kind/SubKind/Algorithm`、`AlgorithmFamily` |
| 管理问卷绑定 | `AssessmentModel.Binding`、application service `BindQuestionnaire` |
| 保存模型定义 payload | `DefinitionPayload{Format, Data}` |
| 发布运行读模型 | `application/modelcatalog/publishedmodel.Build` -> `PublishedModelRepository` |
| 提供运行侧读取 | `PublishedModelReader`、`RuntimePublishedCatalog` |
| 支撑 C 端已发布列表 | `PublishedModelLister`、collection REST 读侧 |
| 保留 legacy 医学量表路径 | `scales` collection、scale repository、scale backfill/publisher |

---

## 4. 模块不负责什么

| 不负责 | 应归属 |
| ------ | ------ |
| 问卷题目、选项、跳题设计 | questionnaire / survey |
| 用户答卷保存 | survey / answersheet |
| 计算某次测评结果 | evaluation |
| 生成 `FactorScore`、`NormScore`、`ConclusionResult` | evaluation |
| 生成最终报告实例 | interpretation / report |
| C 端入口编排和展示策略 | collection-server / application query |
| 统计聚合 | statistics |

## 5. 输入和输出

输入分两类。

配置命令：

```text
CreateAssessmentModel
UpdateBasicInfo
BindQuestionnaire
UpdateDefinition
Validate
Publish
Unpublish
Archive
Delete
```

依赖资料：

```text
QuestionnaireCode
QuestionnaireVersion
Question definitions
Option score definitions
Norm data
Product / Identity options
```

输出：

```text
AssessmentModel draft
Published model snapshot
Published model list
Questionnaire binding resolution
Product / Identity / Family options
Cache invalidation signals
```

## 6. 上下游关系

```text
questionnaire / survey
  -> provides questionnaire_code + questionnaire_version + questions

modelcatalog
  -> owns AssessmentModel draft
  -> publishes PublishedModel / target AssessmentSnapshot

evaluation
  -> consumes published snapshot + AnswerSheet
  -> produces FactorScore / NormScore / ConclusionResult

interpretation / report
  -> consumes EvaluationResult + model report mapping
  -> produces Report
```

当前运行读取入口：

- `infra/ruleset.RuntimePublishedCatalog`
- `infra/modelcatalog.PublishedStore`
- `infra/evaluationinput.PublishedTypologyCatalog`
- `infra/evaluationinput.PublishedBehavioralRatingCatalog`
- `infra/evaluationinput.PublishedCognitiveCatalog`

---

## 7. 过渡状态

| 过渡点 | 当前状态 | 目标 |
| ------ | -------- | ---- |
| 包命名 | `publishing` 实际承载核心 `AssessmentModel` | 改为 `assessmentmodel` 或等价语义 |
| 发布模型 | port 名为 `PublishedModel` | 领域语义收口为 `AssessmentSnapshot` |
| 医学量表 | `scoring/definition.MedicalScale` 仍是完整 legacy 聚合 | 迁移到 `AssessmentModel + Definition + MeasureSpec + RiskConclusion` |
| Factor | `factor`、`scoring/definition.Factor`、`scoring/snapshot.FactorSnapshot` 并存 | 收口为通用测量节点 + 执行视图 adapter |
| norming/taskperformance/typology | 当前更像 payload adapter | 拆成 `Norm`、`Conclusion` 和 compatibility adapter |
