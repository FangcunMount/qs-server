# ModelCatalog 模块设计

## 模块定位

ModelCatalog 管理“某套问卷应如何被计分、校准、解释和呈现”的模型资产；它不保存答卷、不计算某次测评结果，也不生成最终报告实例。

模型身份只接受四种 canonical kind：`scale`、`typology`、`behavioral_rating`、`cognitive`。`ProductChannel` 是产品目录分类，不能作为运行时 evaluator 路由依据。

## 分层

| 层 | 位置 | 职责 |
| --- | --- | --- |
| Domain | `internal/apiserver/domain/modelcatalog` | `AssessmentModel`、`DefinitionV2`、测量/校准/结论/报告映射规则 |
| Application | `internal/apiserver/application/modelcatalog` | actor-oriented use case、授权、生命周期和发布编排 |
| Payload port | `internal/apiserver/port/modelcatalog/payload/*` | published JSON 与运行 DTO 的编解码和投影 |
| Persistence | `internal/apiserver/infra/mongo/modelcatalog` | draft、published model、norm 的 Mongo 映射 |
| Runtime | `infra/evaluationinput`、`infra/ruleset` | 从 published `DefinitionV2` 构造执行输入 |

`domain/modelcatalog` 不承载 family command service、scale authoring aggregate 或 typology runtime DTO。它们已经分别收口为 application use case 或 payload port。

## 应用服务

所有 command/query 接收 transport-neutral `ActorContext`，并经 `Authorizer` 校验 capability 与组织范围。

| 服务 | 主要行为 | Action |
| --- | --- | --- |
| `CatalogManagementService` | 创建、基本信息、问卷绑定、归档、删除、问卷版本同步 | `manage_catalog` |
| `DefinitionAuthoringService` | 读、保存、校验、预览、申请 Definition 编码 | `edit_definition` |
| `PublicationService` | 发布、下架、失败补偿、后置 effect | `publish_catalog` |
| `CatalogQueryService` | draft/published 目录、选项、热榜、二维码 | `read_catalog` |
| `PublishedModelResolver` | 受信任服务主体按 ref 或问卷解析 published model | `resolve_published` |

模型差异仅由 `definition.Registry` 的四个策略处理。生命周期副作用由独立的 `LifecycleEffectsRegistry` 按 identity 注册，不能重新变成 family service 的 publish/lifecycle 分支。

## 外部边界

- 后台 REST 统一为 `/api/v1/assessment-models`；定义编辑整体读写 canonical `DefinitionV2`。
- `AssessmentModelCatalogService` 是唯一模型目录 gRPC，只返回已发布模型及其 `definition_json`。
- collection 保留 `/api/v1/typology-models` 作为业务 BFF facade。它通过通用 catalog gRPC 获取 `DefinitionV2`，再投影为前台人格目录响应；不存在专用 typology gRPC。
- published payload 的格式和 bytes 是兼容 wire artifact。领域、发布校验、运行时路由不得解析它来补齐语义。
- plan、evaluation、notification 等下游只读取 published model；计划创建只能绑定带有 `DefinitionV2` 的已发布 scale，不能借由 draft 模型校验或补齐标题。
