# 模型版本与兼容策略

## 1. 版本原则

Assessment Model 必须版本化。原因是 Evaluation 和 InterpretReport 都要能解释历史结果：

- 当时使用的是哪个模型。
- 当时模型 payload 是什么。
- 当时分值区间、因子和规则是什么。
- 当时报告层如何识别模型身份。

---

## 2. 快照优先

| 场景 | 策略 |
| ---- | ---- |
| 新执行 | 读取发布态快照 |
| 历史追溯 | 使用执行记录中的快照引用 |
| 修改规则 | 发布新版本 |
| 废弃规则 | 下架或归档模型版本，不改写历史快照 |

---

## 3. 兼容注册名

容器注册中 `assessmentmodel` 兼容：

```text
scale
personalitymodel
```

这只是兼容旧注册名或旧能力路径，不代表 `scale/personalitymodel` 仍是独立核心模块。

---

## 4. 文档兼容策略

- 新文档统一写 `assessment-model`。
- 代码引用处保留 `assessmentmodel`。
- 提到旧 Scale 时，写成“医学量表模型资产”。
- 提到旧 Personality Model 时，写成“人格模型资产”。
