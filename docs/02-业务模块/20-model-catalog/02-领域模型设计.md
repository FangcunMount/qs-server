# ModelCatalog 领域模型设计

**本文回答**：当前领域模型是什么、目标领域模型是什么，以及二者如何过渡。

---

## 1. 当前领域模型地图

| 当前对象 | 位置 | 当前职责 | 设计判断 |
| -------- | ---- | -------- | -------- |
| `AssessmentModel` | `domain/modelcatalog/assessmentmodel` | 统一草稿模型聚合，承载身份、产品分类、绑定、定义、生命周期和内部版本 | 已是唯一后台测评模型配置聚合 |
| `PublishedModel` | `port/modelcatalog` | 已发布运行记录，供查询、缓存、evaluation 读取 | 目标语义是 `AssessmentSnapshot` |
| `ProductChannel` | `domain/modelcatalog/binding` | 产品分类：医学量表、人格测评、行为能力 | 不应用作运行时路由依据 |
| `Kind/SubKind/Algorithm` | `domain/modelcatalog/binding` | 模型算法身份 | 目标语义可收口为 `Identity` |
| `Family` / `AlgorithmFamily` | `domain/modelcatalog/identity` | 从身份或决策语义派生的运行机制 | `AlgorithmFamily` 作为兼容语义由根门面 re-export |
| `factor.FactorSnapshot` | `domain/modelcatalog/factor` | 跨算法的 Factor 快照和通用构件 | 目标语义是通用 `Factor` |
| `MedicalScale` | `domain/modelcatalog/scoring/definition` | 旧医学量表完整聚合，带自己的 factor、version、lifecycle、repository | legacy authoring model |
| `norming` snapshot | `domain/modelcatalog/norming` | behavioral_rating / Brief-2 payload decode 和常模 metadata adapter | 不是独立 `Norm` 领域模型 |
| `taskperformance` snapshot | `domain/modelcatalog/taskperformance` | cognitive / SPM payload decode 和 task-set metadata adapter | 不是独立 task measurement 模型 |
| `typology.Payload` | `domain/modelcatalog/typology` | 人格 payload、runtime spec、report spec、legacy MBTI/SBTI 转换 | 应拆成 `TypeConclusion` + profile / adapter |

---

## 2. 目标模型结构

```text
AssessmentModel
└── Definition
    ├── MeasureSpec
    │   ├── Factor[]
    │   ├── FactorGraph
    │   └── Scoring
    ├── Calibration
    │   └── NormRef[]
    ├── Conclusion
    │   ├── RiskConclusion
    │   ├── TypeConclusion
    │   ├── NormConclusion
    │   └── AbilityConclusion
    ├── Outcome[]
    └── ReportMap

AssessmentModel --publish--> AssessmentSnapshot
AssessmentSnapshot --uses--> Definition
```

---

## 3. 核心模型

| 模型 | 类型 | 含义 |
| ---- | ---- | ---- |
| `AssessmentModel` | 聚合根 | 后台测评模型配置。承载产品、算法身份、问卷绑定、生命周期、内部修订号 |
| `AssessmentSnapshot` | 发布态快照 | 针对某个问卷版本发布后的不可变快照，用于查询、缓存、执行 |
| `Definition` | 值对象 | 测评模型定义主体，组合测量、校准、结论、报告映射 |

`AssessmentSnapshot` 是发布态读模型 / 值对象，不是第二个聚合根。

---

## 4. Definition 四层

| 层 | 模型 | 回答的问题 |
| -- | ---- | ---------- |
| 测量层 | `MeasureSpec`、`Factor`、`Scoring`、`FactorGraph` | 测什么，以及如何从题目得到维度分 |
| 校准层 | `Calibration`、`Norm`、`NormRef` | 测量结果是否需要常模或其他资料转换 |
| 结论层 | `Conclusion`、`RiskConclusion`、`TypeConclusion`、`NormConclusion`、`AbilityConclusion`、`Outcome` | 这些测量结果意味着什么 |
| 报告映射层 | `ReportMap` | 如何把模型配置和 evaluation 结果映射到报告展示 |

---

## 5. Factor 设计

`Factor` 是 ModelCatalog 的通用测量节点。

```text
Factor = 测评维度 / trait / score node / task set / ability domain
```

Factor 可以表达：

- 维度编码和名称；
- 题目归属；
- 分数聚合策略；
- 父子层级；
- 总分、指数、效度、任务集、能力域等 role；
- 分类、解释、常模引用。

认知任务也先用 `Factor` 表达，不单独建 TaskMeasurement。只有未来出现 Factor 无法表达的复杂任务执行数据时，再考虑新增独立抽象。

---

## 6. Conclusion 设计

`Conclusion` 回答“这些测量结果意味着什么”。

| 结论类型 | 典型产品 | 输入 | 输出 |
| -------- | -------- | ---- | ---- |
| `RiskConclusion` | 医学量表 | total score / factor score | 风险等级、解释、建议 |
| `TypeConclusion` | 人格测评 | factor scores / factor vector | 类型、模式、trait profile |
| `NormConclusion` | 行为能力 | factor score + norm score | 常模等级、百分位解释 |
| `AbilityConclusion` | 认知任务 | task_set / ability_domain factor score | 能力等级、能力解释 |

`FactorClassificationModel` 不再作为独立核心模型出现。人格分类是 `TypeConclusion` 的配置策略。

---

## 7. 配置侧与结果侧

| ModelCatalog 配置侧 | Evaluation 结果侧 |
| ------------------- | ----------------- |
| `Factor` | `FactorScore` |
| `NormRef` / `Norm` | `NormScore` |
| `Conclusion` | `ConclusionResult` |
| `Outcome` | `OutcomeResult` |
| `ReportMap` | `ReportData` / Report |

ModelCatalog 不消费用户答卷，不产出用户测评结果。
