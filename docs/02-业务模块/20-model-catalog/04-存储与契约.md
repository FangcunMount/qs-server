# ModelCatalog 存储与契约

## Mongo 事实

| Collection | 内容 | 读取方 |
| --- | --- | --- |
| `assessment_models` | editable `AssessmentModel` 和 `definition_v2` | 管理、定义设计、目录查询 |
| `published_assessment_models` | immutable `AssessmentSnapshot`、`definition_v2` 和 wire payload | 发布目录、resolver、evaluation、ruleset、collection |
| `assessment_norms` | versioned `Norm` lookup/band 数据 | 发布校验和运行时常模解析 |

旧 `scales` collection 不再属于生产 ModelCatalog 的读写路径。历史数据迁移由 one-off script 处理；脚本若需 BSON compatibility struct，必须私有定义，不能重新依赖旧 domain/application 包。

## published snapshot

`PublishedModel` 是 `AssessmentSnapshot` 的 port 兼容别名。它的事实源是：

```text
identity + questionnaire binding + DefinitionV2 + DecisionKind
```

`payload_format` 和 `payload` 保留为外部 wire/runtime DTO 契约。它们不用于 runtime semantic read，也不用于目录或发布校验。`source` 不承载 legacy scale binding 语义。

## Ports

| Port | 用途 |
| --- | --- |
| `ModelRepository` | draft aggregate 读写 |
| `PublishedModelRepository` | published snapshot 写入、删除与列表 |
| `PublishedModelReader` / `PublishedModelLister` | published-only 读取 |
| `NormRepository` | versioned norm upsert 与解析 |
| `port/modelcatalog/payload/*` | scale、behavioral、cognitive、typology 的 JSON/runtime DTO |

## Transport contract

- 后台 REST：`/api/v1/assessment-models`。`PUT /{code}/definition` 整体保存 canonical Definition JSON。
- gRPC：`AssessmentModelCatalogService` 是唯一模型目录服务，详情的 `definition_json` 是 canonical Definition JSON。
- collection REST：`/api/v1/assessment-models` 是通用已发布目录；`/api/v1/typology-models` 是其业务 BFF 投影。
- 旧的 scale REST/gRPC 与专用 typology proto 已删除。

任何新消费者都必须使用通用 catalog contract 或 trusted `PublishedModelResolver`，不能新增 payload semantic fallback。
