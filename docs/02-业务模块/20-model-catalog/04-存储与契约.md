# ModelCatalog 存储与契约

**本文回答**：ModelCatalog 当前有哪些存储事实、port 契约和兼容字段。

---

## 1. 存储总览

| Collection | 当前用途 | 目标定位 |
| ---------- | -------- | -------- |
| `assessment_models` | 新后台测评模型配置 | 存储 `AssessmentModel` 聚合状态 |
| `published_assessment_models` | 发布后的运行记录 | 存储目标语义上的 `AssessmentSnapshot` |
| `scales` | 旧医学量表 head / published snapshot 双记录 | legacy medical-scale authoring store |

---

## 2. `assessment_models`

Mongo PO：

```text
infra/mongo/modelcatalog.AssessmentModelPO
```

关键字段：

```text
code
product_channel
kind
sub_kind
algorithm
title
description
category
tags
status
questionnaire_code
questionnaire_version
definition_payload_format
definition_payload
version
published_at
archived_at
```

当前 `version` 是草稿聚合内部版本 / 乐观更新依据。目标语义上它应收口为 `Revision` / `ConfigRevision`，不是测评业务版本。

---

## 3. `published_assessment_models`

Mongo PO：

```text
infra/mongo/modelcatalog.PublishedAssessmentModelPO
```

关键字段：

```text
schema_version
payload_format
model_product_channel
model_kind
model_sub_kind
model_algorithm
model_code
model_version
title
status
decision_kind
questionnaire_code
questionnaire_version
source
payload
published_at
```

目标语义：

- 这是 `AssessmentSnapshot` 的持久化形态。
- `model_code + questionnaire_code + questionnaire_version` 是目标业务唯一性。
- `model_version` 短期保留为兼容字段；目标上不再表达独立测评业务版本。
- `payload` 当前仍是运行 payload bytes，后续可逐步收口为 `Definition` 的发布形态。

当前 upsert 注意点：

- 当前 upsert filter 不包含 `model_version`。
- 这符合“同一模型 code 当前只有一个 active published row”的现状。
- 后续如果按问卷版本建立业务唯一性，需要同步调整 repository filter、索引和缓存 key。

---

## 4. port 契约

草稿仓储：

```text
port.ModelRepository
```

关键方法：

```text
Create
Update
FindByCode
List
Delete
```

发布模型仓储：

```text
port.PublishedModelRepository
```

关键方法：

```text
Save
FindPublishedByModelCode
FindLatestPublishedByModelCode
FindPublishedByModelCodeVersion
ListPublished
DeletePublished
```

运行读取：

```text
port.PublishedModelReader
port.PublishedModelLister
port.QuestionnaireResolver
port.Catalog
```

重要约束：

- Draft `ModelRepository` 不应用于 evaluation 执行。
- Evaluation 和 C 端已发布查询应通过 published reader / lister。
- `GetPublishedModelByRef` 当前要求 version 必填；这是现有兼容契约，后续版本语义迁移时需要单独重构。

---

## 5. `scales`

Mongo PO：

```text
infra/mongo/scale.ScalePO
```

关键字段：

```text
code
scale_version
record_role
is_active_published
questionnaire_code
questionnaire_version
status
factors
```

目标定位：

- `scales` 是 legacy medical-scale authoring store。
- 它保留历史兼容能力。
- 目标模型不再从 `scales` 推导，而应从 `AssessmentModel + Definition + MeasureSpec + RiskConclusion` 推导。
