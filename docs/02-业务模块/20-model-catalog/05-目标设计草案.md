# ModelCatalog 终局设计

本文件记录已经落地的设计约束，而不是待实施草案。

1. `AssessmentModel + DefinitionV2` 是唯一可编辑模型事实。
2. `AssessmentSnapshot` 是 immutable published read model；它不能被当成第二个 authoring aggregate。
3. `DefinitionV2` 分为 Measure、Calibration、Conclusion、Outcome/ReportMap 四层。
4. `Factor` 只表示测量节点身份。题目来源和组合计分属于 `Scoring`，层级属于 `FactorGraph`，常模引用属于 `Calibration`，解释和分类属于 `Conclusion`。
5. `Norm` 是独立版本化资料，`NormRef` 仅绑定 factor 和 norm version。
6. 模型 identity 只接受 `scale`、`typology`、`behavioral_rating`、`cognitive`；`personality` 不是 ModelCatalog kind 或 product-channel alias。
7. application 以行为人/用例划分为五类服务；模型差异只存在于 `definition.Registry` strategy 和 payload adapter。
8. publish/runtime 只从 `DefinitionV2` 推导语义。payload 是 wire artifact，不是 fallback 事实源。
9. `AssessmentModelCatalogService` 是唯一内部模型目录 gRPC；collection 的 `/typology-models` 是 generic catalog 的 BFF 投影。
10. 旧 scale collection、旧 family command service、old option router、special typology gRPC 不得重新引入。

上述不变量由 application/domain architecture tests 保护。新增模型能力应先扩展 Definition 和 strategy，再增加 payload projection；不得重新添加按 code 的 application hard-code 分支。
