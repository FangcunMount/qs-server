# ModelCatalog 关键链路

## 1. 定义设计

```text
REST/gRPC actor
  -> DefinitionAuthoringService
  -> Authorizer(edit_definition)
  -> ModelRepository.FindByCode
  -> definition.Registry.Resolve(identity)
  -> Validate DefinitionV2
  -> strategy.BuildSnapshotPayload (wire projection)
  -> AssessmentModel.UpdateDefinitionWithV2
  -> assessment_models
```

定义编辑主输入和主校验都是 `DefinitionV2`。payload 只在保存后由 strategy 生成，以维持 published wire contract；不能作为 authoring 语义输入或校验来源。

## 2. 目录管理与发布

```text
CatalogManagementService
  -> Create / UpdateBasicInfo / BindQuestionnaire / Archive / Delete

PublicationService
  -> questionnaire binding policy
  -> definition.Registry strategy validation
  -> publication.Publisher
  -> PublishedModelRepository.Save
  -> lifecycle effects after persistence
```

发布写入不可变 snapshot，同时更新 draft 生命周期。draft 持久化失败时 publisher 删除刚创建的 snapshot；缓存和事件等 effect 只在持久化成功后 best-effort 触发，不反转发布结果。

## 3. 运行时解析

```text
trusted service actor
  -> PublishedModelResolver
  -> PublishedModelReader
  -> require DefinitionV2
  -> evaluationinput / ruleset family adapter
  -> execution DTO
```

运行时只读 `published_assessment_models` 的 `DefinitionV2` 和 `NormRepository`。缺少 `DefinitionV2` 或被引用常模时返回配置错误；不读取 draft、不读取旧 collection，也不解析 payload 补齐配置。

## 4. collection 人格目录

```text
GET /typology-models
  -> collection typology BFF/cache
  -> AssessmentModelCatalogService
  -> definition_json
  -> DefinitionV2 + payload/typology presentation projector
  -> existing typology REST response
```

`/typology-models` 保留业务 REST response 和缓存边界，但它不是独立模型目录或专用 gRPC consumer。投影读取 `RuntimeSpecFromDefinition`，而不是 payload 的 legacy flat dimension 字段。

## 5. 四类 strategy

| Kind | 领域语义 | 发布 payload 投影 |
| --- | --- | --- |
| `scale` | Measure + RiskConclusion | `payload/scale` |
| `behavioral_rating` | Measure + Calibration + NormConclusion | `payload/behavioral` |
| `cognitive` | Measure + Calibration + AbilityConclusion | `payload/cognitive` |
| `typology` | Measure graph + TypeConclusion + Outcome + ReportMap | `payload/typology` |
