# 解释模型设计

## 1. 业务目标

把结构化测评结果转换为可以被用户理解、被从业者解释、被查询端展示的报告内容。

---

## 2. 解释内容

| 内容 | 说明 |
| ---- | ---- |
| 分数区间解释 | 按总分或维度分匹配说明 |
| 因子解释 | 对每个因子分给出含义 |
| 风险提示 | 根据等级或阈值生成风险提示 |
| 建议文案 | 根据结果生成行动建议 |
| 报告模板 | 组织 section、标题和排序 |

---

## 3. 设计原则

- 解释模型消费 `EvaluationResult`，不重新计分。
- 报告模板组织展示，不修改执行结果。
- 同一执行结果可按模型身份选择不同 builder。
- 报告内容应能追溯模型身份和执行结果。

---

## 4. 当前代码形态

当前代码没有独立 `interpretationmodel` 包，解释能力主要位于：

- `internal/apiserver/domain/interpretation`
- `internal/apiserver/domain/interpretation/scoring`
- `internal/apiserver/domain/interpretation/typology`
- `internal/apiserver/container/modules/interpretation`

文档先统一业务语言，不要求立即重命名代码包。
