# Interpretation Model 模块定位与边界

## 1. 模块定位

Interpretation Model 是解释模型与报告产出层。它把 Evaluation 生成的结构化结果转换成用户、从业者或运营端可理解的报告。

当前代码包仍叫 `interpretation`，文档业务语言使用 `interpretation`。

**当前实现**：报告 Builder 与持久化在 `container/modules/interpretation` 装配，但解读执行与报告生成编排仍在 `evaluation/execute` + `evaluation/result/writer` 内同步完成。目标是将 Interpretation 编排迁入本模块（见 [`04-报告生成链路.md`](./04-报告生成链路.md) §0）。

---

## 2. 负责范围

| 范围 | 说明 |
| ---- | ---- |
| 解释模型 | 分数区间、因子解释、风险提示、建议文案 |
| 报告模型 | 报告 section、模板、输出结构 |
| 报告构建 | Builder registry、score / personality adapter |
| 报告实例 | `InterpretReport` 聚合与持久化 |
| 报告事件 | `report.generated` |

---

## 3. 不负责范围

| 不负责 | 归属模块 |
| ------ | -------- |
| 作答提交 | `survey` |
| 模型资产发布 | `model-catalog` |
| 测评执行状态机 | `evaluation` |
| 周期任务调度 | `plan` |
| 统计聚合 | `statistics` |

---

## 4. 边界反例

- `InterpretReport` 不是 `Evaluation`。
- `InterpretationModel` 不是 `AssessmentModel`。
- 报告生成失败不应改写答卷事实。
- 报告查询不是统计查询。
