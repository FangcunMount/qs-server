# 解释适配器与模板机制

## 1. 业务目标

用可扩展的 builder 和 adapter 支持不同测评模型的报告生成，同时保持 `InterpretReport` 的统一输出。

---

## 2. 适配类型

| 类型 | 说明 |
| ---- | ---- |
| Score adapter | 适配分数型或医学量表报告 |
| Personality adapter | 适配人格测评报告 |
| Generic section | 复用通用报告 section 结构 |
| Suggestion strategy | 生成建议和行动提示 |

---

## 3. 模板结构

| 模板部分 | 说明 |
| -------- | ---- |
| Overview | 总览、等级、摘要 |
| Dimension Sections | 因子、维度或人格特质 |
| Risk Section | 风险提示 |
| Suggestion Section | 建议文案 |
| Metadata | 模型身份、生成时间、追溯信息 |

---

## 4. 扩展规则

- 新报告类型优先新增 builder / adapter，不改 Evaluation 主链路。
- 模板只组织表达，不承担计分。
- Builder 输入应来自 EvaluationResult 和模型身份。
- Builder 可以使用风险等级、分类和维度分等评估事实，但 `conclusion / suggestion`、章节和受众化文案由本模块产生。
- Report 重建只重跑 Interpretation，不调用 Calculation，不覆盖 `assessment_score`，不新增 EvaluationRun。
- 输出应收敛到统一 `InterpretReport`。
