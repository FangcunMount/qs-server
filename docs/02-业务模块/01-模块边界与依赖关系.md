# 模块边界与依赖关系

## 1. 允许的协作

- Survey 通过提交事件把 AnswerSheet 事实交给异步链。
- Evaluation 通过 port 读取问卷与已发布模型运行时视图。
- Interpretation 只读 Evaluation Outcome，并独立提交报告事实。
- Actor 提供访问范围和参与者上下文，不替业务模块执行状态迁移。
- Plan 生成/管理任务，并通过明确入口关联测评过程。
- Statistics 读取或投影其它模块事实，不成为写侧依赖。

## 2. 禁止的协作

- collection/worker 直接写业务 repository；
- Evaluation 读取可编辑草稿；
- Interpretation 回写 Evaluation 状态；
- Statistics 投影反向成为业务决策的唯一真值；
- 模块通过 infra 具体实现互相调用而绕过 port/application 边界。
