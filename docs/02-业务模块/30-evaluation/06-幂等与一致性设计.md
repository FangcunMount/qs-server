# 幂等与一致性设计

## 1. 设计目标

Evaluation 通常由事件驱动，必须处理重复消费、部分失败、下游异步和历史追溯。

---

## 2. 幂等边界

| 边界 | 幂等依据 |
| ---- | -------- |
| 从答卷创建测评 | `AnswerSheetID`、模型绑定、任务上下文 |
| 执行模型快照 | `AssessmentID`、`SnapshotID`、执行尝试 |
| 写入结果 | `EvaluationID` 或 `AssessmentID` |
| 发布事件 | outbox 可靠出站语义 |

---

## 3. 一致性原则

| 场景 | 原则 |
| ---- | ---- |
| 答卷已提交但执行失败 | 不回滚 `AnswerSheet` |
| 执行完成但报告未生成 | 保留 `EvaluationResult`，由报告层补偿 |
| 执行完成但统计未更新 | 主链路成功，统计最终一致 |
| 模型配置变更 | 历史执行仍引用旧快照 |

---

## 4. 重试边界

Evaluation 可以重试执行失败，但不应重试 Survey 答卷提交，也不应直接重试报告渲染。跨模块补偿通过事件、任务或运维入口协调。
