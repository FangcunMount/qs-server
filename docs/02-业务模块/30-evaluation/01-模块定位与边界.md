# Evaluation 模块定位与边界

## 1. 模块定位

Evaluation 是测评执行层。它接收作答事实和模型快照，完成计分、因子计算、等级判定和结构化结果生成。

---

## 2. 负责范围

| 范围 | 说明 |
| ---- | ---- |
| 测评实例 | `Assessment` 的创建和生命周期 |
| 执行尝试 | `EvaluationRun`、错误、重试 |
| 模型执行 | 加载快照，选择执行器，运行规则 |
| 结果生成 | 总分、因子分、等级、原始结果 |
| 执行事件 | 提交、完成、失败等事件 |

---

## 3. 不负责范围

| 不负责 | 归属模块 |
| ------ | -------- |
| 问卷定义 | `survey` |
| 模型资产发布 | `model-catalog` |
| 报告模板和解释文案 | `interpretation` |
| 周期任务调度 | `plan` |
| 统计读模型 | `statistics` |

---

## 4. 边界反例

- `Evaluation` 不是 `AnswerSheet`。
- `EvaluationResult` 不是 `InterpretReport`。
- `AssessmentModelSnapshot` 不是 Evaluation 的内部可变配置。
- `assessment.interpreted` 不是统计指标更新完成。
