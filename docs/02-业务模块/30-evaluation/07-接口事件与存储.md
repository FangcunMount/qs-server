# Evaluation 接口事件与存储

## 1. 接口边界

Evaluation 暴露测评创建、执行、查询和内部回调能力。接口契约以 OpenAPI / gRPC 为准，本文只描述业务含义。

---

## 2. 事件边界

| 事件 | Delivery | 说明 |
| ---- | -------- | ---- |
| `assessment.submitted` | `durable_outbox` | 测评进入执行链路 |
| `assessment.interpreted` | `durable_outbox` | 测评结构化结果完成 |
| `assessment.failed` | `durable_outbox` | 测评执行失败 |

---

## 3. 存储边界

Evaluation 保存测评实例、执行状态、执行结果和失败原因。它不保存问卷定义、模型草稿、最终报告模板或统计聚合。

---

## 4. 一致性

| 写入 | 要求 |
| ---- | ---- |
| 状态变更 | 按领域方法迁移，避免越级改状态 |
| 结果写入 | 与执行状态保持一致 |
| 事件出站 | 与状态 / 结果持久化可靠衔接 |
