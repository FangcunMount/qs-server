# 统计指标模型

## 1. 业务目标

把业务链路中的事实转换成运营、服务质量和计划完成情况可查询的指标。

---

## 2. 指标分组

| 指标组 | 来源 | 示例 |
| ------ | ---- | ---- |
| Content 指标 | assessment 事实/content daily | 形成 assessment 数、evaluated 数、完成率 |
| Overview 指标 | org snapshot/content daily/主事实 | 内容数、累计与今日答卷提交数 |
| Assessment Service | journey daily | 答卷提交、测评创建、报告生成 |
| Plan Activity | plan daily | 创建、打开、完成、过期 |
| Plan Fulfillment | 任务事实 | 应到期、按时完成、逾期、完成率 |
| Journey 指标 | behavior footprint | 入口打开、接入确认、建档、关系建立 |

---

## 3. 时间窗口

常见窗口包括：

- today
- 7d
- 30d

具体支持范围以代码和接口契约为准。

---

## 4. 关键规则

- 指标必须能回到事件或读模型来源。
- 不同口径要明确分母和时间窗口。
- 内容完成数仅统计 `status=evaluated`，不从 Mongo 答卷数量推断。
- 类型、编码和机构共同定义内容身份。
- 统计指标允许延迟，不应用作主链路成功条件。
