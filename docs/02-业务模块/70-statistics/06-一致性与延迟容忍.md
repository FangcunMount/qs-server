# 一致性与延迟容忍

## 1. 设计目标

Statistics 接受最终一致，因为它服务读侧查询和运营观察，不参与核心业务事务提交。

---

## 2. 一致性表

| 场景 | 主事实 | Statistics 行为 |
| ---- | ------ | --------------- |
| 答卷提交 | `survey` | 异步增加提交指标 |
| 测评完成 | `evaluation` | 异步更新完成率 |
| 报告生成 | `interpretation` | 异步更新报告指标 |
| 任务完成 | `plan` | 异步更新计划完成率 |

---

## 3. 延迟容忍

| 延迟 | 处理 |
| ---- | ---- |
| 秒级延迟 | 查询侧可接受 |
| 分钟级延迟 | 需要排查 projector、队列或缓存 |
| 长时间缺口 | 需要重放事件或重建读模型 |

---

## 4. 禁止事项

- 不要让核心业务事务等待统计写入。
- 不要用统计数据判断某个测评是否真实完成。
- 不要让统计投影反向修改 Plan 或 Evaluation 状态。
