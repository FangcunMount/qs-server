# Statistics 模块定位与边界

## 1. 模块定位

Statistics 是读侧统计与行为投影模块。它把业务事件和读模型聚合成查询视图，帮助运营后台和数据看板理解系统运行。

---

## 2. 负责范围

| 范围 | 说明 |
| ---- | ---- |
| 行为投影 | 接入、建档、答卷、测评、报告等 footprint |
| 指标聚合 | journey、content、plan、organization snapshot |
| 查询视图 | 机构、从业者、入口、周期、统一内容统计 |
| 缓存治理 | Overview 查询缓存和读模型刷新 |

---

## 3. 不负责范围

| 不负责 | 归属模块 |
| ------ | -------- |
| 答卷事实 | `survey` |
| 执行状态 | `evaluation` |
| 报告事实 | `interpretation` |
| 任务主状态 | `plan` |
| 业务身份主模型 | `actor` |

---

## 4. 边界反例

- `StatisticProjection` 不是业务主事实。
- `PlanMetric` 不是 `AssessmentTask`。
- `AssessmentMetric` 不是 `EvaluationResult`。
- 统计延迟不代表主链路失败。
