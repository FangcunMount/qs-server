# Statistics 领域模型

Statistics 是可重建的读侧统计模块，不定义通用 `StatisticProjection` 聚合根，也不保存 Survey、Evaluation、Interpretation、Plan 的主事实。

## 当前模型

| 模型 | 作用 | 主要身份 |
| ---- | ---- | -------- |
| `BehaviorFootprint` | 幂等记录接入旅程事件 | `org_id + event_name + source_type + source_id` |
| `StatisticsJourneyDaily` | 机构、从业者、入口的日旅程聚合 | `org_id + subject_type + subject_id + stat_date` |
| `StatisticsContentDaily` | questionnaire/scale 内容使用与完成聚合 | `org_id + content_type + content_code + stat_date` |
| `StatisticsPlanDaily` | 计划活动与履约日聚合 | `org_id + plan_id + stat_date` |
| `StatisticsOrgSnapshot` | 机构累计快照 | `org_id` |
| `StatisticsSyncWatermark` | 各扫描源的增量水位 | `org_id + source` |
| `StatisticsJourneyPending` | 尚待重算的投影窗口 | 机构、日期和状态 |

## 查询值对象

- `OrganizationOverview`：机构累计规模、内容数和答卷提交数。
- `AccessFunnelStatistics`、`AssessmentServiceStatistics`：窗口与趋势。
- `ClinicianStatistics`、`AssessmentEntryStatistics`：从业者和入口统计。
- `ContentBatchStatisticsItem`：带 `content_type` 的统一内容统计。
- `PlanDomainStatistics`：只由 `activity` 与 `fulfillment` 组成。

完成率是确定性函数：分母为零时返回 0，否则为 `completed / total * 100`。模块没有无状态 Validator 或 TrendAnalyzer 对象。

## 边界

- assessment 完成口径只认 `status=evaluated`。
- questionnaire 与 scale 即使 code 相同，也按 `(content_type, content_code)` 分开。
- 所有读写均带机构条件；投影缺失时只能回到机构范围内的主事实查询。
- 投影延迟不改变主业务事实，投影也不得反向驱动业务状态。
