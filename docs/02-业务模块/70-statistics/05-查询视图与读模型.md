# 查询视图与读模型

## 1. 业务目标

为运营后台、小程序或数据看板提供低成本、面向查询的统计视图。

---

## 2. 查询视图

| 视图 | 说明 |
| ---- | ---- |
| 机构概览 | 机构范围内人员、测评、报告、计划概览 |
| 接入漏斗 | 从入口打开到建档、关系建立、答卷提交的转化 |
| 测评服务 | 答卷提交、测评创建、报告生成、失败情况 |
| 计划统计 | 计划、任务、完成、过期 |
| 受试者维度 | 单个受试者的测评和计划概览 |

---

## 3. 读模型原则

- 为查询而建，不替代写模型。
- 可以包含冗余字段。
- 应记录更新时间和统计窗口。
- 口径变化需要同步文档和接口说明。

---

## 4. 缓存边界

缓存可以提升查询性能，但不改变统计事实。缓存失效或延迟不应影响主业务链路。
