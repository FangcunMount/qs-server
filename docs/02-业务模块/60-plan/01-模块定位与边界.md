# Plan 模块定位与边界

## 1. 模块定位

Plan 是测评计划与任务编排层。它管理从计划到任务的生命周期，决定什么时候开放入口、什么时候完成或关闭任务。

---

## 2. 负责范围

| 范围 | 说明 |
| ---- | ---- |
| 计划 | `AssessmentPlan` 的创建和生命周期 |
| 任务 | `AssessmentTask` 的生成、开放、完成、过期、取消 |
| 周期 | 周期规则和触发时间 |
| 协作引用 | 记录任务和答卷、测评、报告的关联 |

---

## 3. 不负责范围

| 不负责 | 归属模块 |
| ------ | -------- |
| 答案校验 | `survey` |
| 计分执行 | `evaluation` |
| 报告生成 | `interpretation-model` |
| 指标聚合 | `statistics` |

---

## 4. 边界反例

- `AssessmentTask` 不是 `AnswerSheet`。
- `AssessmentPlan` 不是 `AssessmentModel`。
- 任务完成不等于报告生成。
- 任务状态不是 Evaluation 状态。
