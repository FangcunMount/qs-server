# Survey 模块定位与边界

## 1. 模块定位

Survey 是作答事实层。它把问卷定义、题目结构和用户提交的答案沉淀成可被后续测评执行消费的 `AnswerSheet`。

Survey 的业务目标不是“完成测评”，而是保证作答事实清晰、可校验、可追溯。

---

## 2. 负责范围

| 范围 | 说明 |
| ---- | ---- |
| 问卷定义 | `Questionnaire`、标题、编码、类型、状态和版本 |
| 题目结构 | 题型、选项、必填、展示和提交规则 |
| 答案结构 | 单题答案、原始值、归一化值 |
| 答卷提交 | 创建并持久化 `AnswerSheet` |
| 提交事件 | 发布 `answersheet.submitted` |

---

## 3. 不负责范围

| 不负责 | 归属模块 |
| ------ | -------- |
| 模型 Kind、规则资产、发布快照 | `model-catalog` |
| 计分、因子计算、等级判定 | `evaluation` |
| 解释文案、报告模板、最终报告 | `interpretation` |
| 周期任务开放、完成、过期 | `plan` |
| 统计聚合、趋势和漏斗 | `statistics` |

---

## 4. 边界反例

- `AnswerSheet` 不是 `AssessmentResult`。
- `Questionnaire` 不是 `AssessmentModel`。
- 题目选项不是计分规则权威。
- 提交成功不是测评完成。
- 答卷状态不是任务状态。
