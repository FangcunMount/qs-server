# Survey 接口事件与存储

## 1. 接口边界

Survey 对外提供问卷查询、问卷维护和答卷提交能力。接口形态以 REST / gRPC 契约为准，文档只描述业务边界。

| 能力 | 业务语义 |
| ---- | -------- |
| 问卷查询 | 获取发布态问卷和题目结构 |
| 问卷维护 | 创建、编辑、发布和归档问卷 |
| 答卷提交 | 校验答案并创建 `AnswerSheet` |

---

## 2. 事件边界

| 事件 | Delivery | 说明 |
| ---- | -------- | ---- |
| `questionnaire.changed` | `best_effort` | 用于缓存和目录刷新 |
| `answersheet.submitted` | `durable_outbox` | 测评执行链路的可靠入口 |

事件契约见 [`configs/events.yaml`](../../../configs/events.yaml)。

---

## 3. 存储边界

Survey 持久化问卷定义和答卷事实。它可以保存提交所需的问卷引用、题目引用和答案值，但不保存执行结果或报告内容。

---

## 4. 一致性

| 场景 | 一致性要求 |
| ---- | ---------- |
| 答卷提交 | `AnswerSheet` 持久化和提交事件必须可靠衔接 |
| 问卷变更 | 发布后的历史版本必须能解释历史答卷 |
| 下游执行 | 下游失败不回滚作答事实 |
