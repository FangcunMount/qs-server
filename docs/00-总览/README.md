## 2. 30 秒结论

`qs-server` 是以 `qs-apiserver` 为主业务中心、由 `collection-server` 和 `qs-worker` 协作的三进程系统。业务事实按 `survey -> modelcatalog -> evaluation -> interpretation` 分层，`actor`、`plan`、`statistics` 提供参与者、编排与读侧支撑。
