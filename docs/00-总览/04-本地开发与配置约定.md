# 本地开发与配置约定

本文档说明如何用 Makefile 选择环境、各进程对应哪些配置文件，以及常用验证方式。细节以仓库内 `Makefile` 与 `configs/*.yaml` 为准。

## 结论要点

- 通过 `ENV=dev`（默认）或 `ENV=prod` 切换配置片段与**对外 HTTP 端口**；同一套源码，不同环境用不同 yaml。
- 三个运行时进程各有一份独立配置：`apiserver`、`collection-server`、`worker`。
- 领域事件路由由全仓共享的 [configs/events.yaml](../../configs/events.yaml) 描述，发布在 `apiserver`，订阅在 `worker`（及文档中声明的其他消费者）。

## 环境与端口

`Makefile` 中约定（节选）：

| 环境变量 | 含义 |
| -------- | ---- |
| `ENV=dev` | 默认；使用 `configs/*.dev.yaml`，apiserver `18082`，collection `18083` |
| `ENV=prod` | 使用 `configs/*.prod.yaml`，apiserver `8081`，collection `8082`（避免与宿主机 IAM 等端口冲突） |

worker 无对外 HTTP 健康检查端口时，以各环境 yaml 内 gRPC/MQ 等配置为准。

启动示例：

```bash
make run-apiserver          # ENV 默认 dev
ENV=prod make run-apiserver
```

## 配置文件对应关系

| 进程 | 开发配置 | 生产配置 |
| ---- | -------- | -------- |
| qs-apiserver | [configs/apiserver.dev.yaml](../../configs/apiserver.dev.yaml) | [configs/apiserver.prod.yaml](../../configs/apiserver.prod.yaml) |
| collection-server | [configs/collection-server.dev.yaml](../../configs/collection-server.dev.yaml) | [configs/collection-server.prod.yaml](../../configs/collection-server.prod.yaml) |
| qs-worker | [configs/worker.dev.yaml](../../configs/worker.dev.yaml) | [configs/worker.prod.yaml](../../configs/worker.prod.yaml) |

其他：

- [configs/events.yaml](../../configs/events.yaml)：事件 Topic、类型与 handler 绑定（与代码中的注册名一致）。
- [configs/seeddata.yaml](../../configs/seeddata.yaml)：`cmd/tools/seeddata` 等辅助工具使用。

## 常用命令与验证

- 构建与运行：`make build`、`make run-all`（见仓库根目录 `Makefile` 的 `help` 目标）。
- 健康检查：`curl http://localhost:18082/healthz`（dev）、`curl http://localhost:18083/healthz`（collection，dev）。
- 容器化本地栈：`build/docker/docker-compose.dev.yml` / `docker-compose.prod.yml`。
