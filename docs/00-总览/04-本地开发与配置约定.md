# 本地开发与配置约定

## 1. 结论

三个进程分别读取自己的配置文件；本地验证应使用仓库 `Makefile` 提供的目标，并避免把开发配置推断为生产事实。

## 2. 配置入口

| 进程 | 开发配置 | 生产配置 |
| --- | --- | --- |
| qs-apiserver | `configs/apiserver.dev.yaml` | `configs/apiserver.prod.yaml` |
| collection-server | `configs/collection-server.dev.yaml` | `configs/collection-server.prod.yaml` |
| qs-worker | `configs/worker.dev.yaml` | `configs/worker.prod.yaml` |

环境变量说明见 `configs/env/README.md`。具体默认值必须回到当前配置核对，文档不复制维护完整参数表。

## 3. 常用验证

```bash
make docs-hygiene
make docs-facts
make test
make lint-boundaries
```

若系统 `go` 版本不匹配，可使用本机已安装的 Go 1.25.12 路径执行定向测试；最终以 `go.mod` 和 CI 环境为准。

## 4. 契约更新

- REST 生成与对比：`make docs-verify`。
- gRPC：从 `api/grpc/proto` 生成，不手改生成代码。
- 事件或信令：同时更新 YAML 契约、代码常量/注册和契约测试。
