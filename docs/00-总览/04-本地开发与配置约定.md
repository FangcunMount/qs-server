# 本地开发与配置约定

**本文回答**：如何用 **`make` + `ENV`** 选用配置与端口、三个进程各自读哪份 yaml、事件与种子数据配置在哪，以及本地如何快速自检服务起来。实现细节以仓库根目录 [Makefile](../../Makefile) 与各 [configs/*.yaml](../../configs/) 为准。

---

## 1. 结论要点

- **`ENV`**：`dev`（默认）或 `prod`，决定 Makefile 选用的 **yaml 路径**以及 **apiserver / collection 的对外 HTTP 端口**（worker 无对外 HTTP 端口约定时，以 yaml 内 gRPC/MQ 为准）。  
- **三进程三配置**：apiserver、collection-server、worker 各有 `*.dev.yaml` / `*.prod.yaml`。  
- **事件全仓一份**：[configs/events.yaml](../../configs/events.yaml) — 发布端（apiserver）与订阅端（worker 等）共用，改事件必须改此文件并对照 handler 注册。  
- **契约**：REST 见 [api/rest/](../../api/rest/)；与端口、TLS 相关的运维说明见 [04-接口与运维/03-部署与端口.md](../04-接口与运维/03-部署与端口.md)。

---

## 2. `Makefile` 与环境变量

| 变量 / 约定 | 含义 |
| ----------- | ---- |
| `ENV=dev`（默认） | `configs/apiserver.dev.yaml`、`collection-server.dev.yaml`、`worker.dev.yaml`；HTTP：**apiserver `18082`**，**collection `18083`** |
| `ENV=prod` | 对应 `*.prod.yaml`；HTTP：**apiserver `8081`**，**collection `8082`**（避免与常见 IAM 端口冲突） |
| `make run-apiserver` 等 | 具体目标以 `make help` 为准 |

**示例**：

```bash
make run-apiserver
ENV=prod make run-collection
make run-all
```

## 4. 常用命令与健康检查

- **构建 / 运行**：`make build`、`make run-all`（完整列表 `make help`）。  
- **HTTP 健康检查（dev 默认端口）**：  
  - `curl -sS http://127.0.0.1:18082/healthz`（apiserver）  
  - `curl -sS http://127.0.0.1:18083/healthz`（collection）  
- **qs-worker**：无对外 HTTP 时，可用 `make health-check`：脚本会 `curl` 上述两服务，并对 **worker** 检查 `tmp/pids/worker.pid` 是否存在且进程存活（与 [Makefile](../../Makefile) 中 `health-check` 目标一致）。  
- **基础设施就绪**：`make check-infra`（全量）或 `make check-mysql` / `check-redis` / `check-mongodb` / `check-nsq`（各依赖调用 [scripts/check-infra.sh](../../scripts/check-infra.sh)）。  
- **容器化本地栈**：[build/docker/docker-compose.dev.yml](../../build/docker/docker-compose.dev.yml)、[build/docker/docker-compose.prod.yml](../../build/docker/docker-compose.prod.yml)。
