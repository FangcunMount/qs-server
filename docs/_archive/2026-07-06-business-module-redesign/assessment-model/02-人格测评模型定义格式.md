# 02-人格测评模型定义格式

## 结论

人格定义权威格式为 **`assessmentmodel.personality.typology.v1`**，payload 内嵌 **`RuntimeSpec`**。前端 `personality_payload_v1` 须映射到此格式，不得另起契约。

## 定义信封

```json
{
  "kind": "personality",
  "sub_kind": "typology",
  "algorithm": "mbti",
  "payload_format": "assessmentmodel.personality.typology.v1",
  "payload": {
    "factor_graph": {},
    "decision": {},
    "special_rules": [],
    "outcome_mapping": {},
    "report": {}
  }
}
```

## RuntimeSpec 结构

定义于 `internal/apiserver/domain/instrument/personality/typology/spec.go`：

| 字段 | 说明 |
|------|------|
| `factor_graph` | 显式因子图（factors + roots） |
| `decision` | 结果决策（pole_composition / nearest_pattern / score_range / trait_profile） |
| `special_rules` | 特殊规则覆盖 |
| `outcome_mapping` | outcome code → 展示/报告映射 |
| `report` | 报告装配规格 |

## algorithm 枚举

`mbti` | `sbti` | `bigfive` | `custom_typology`（与 domain `Algorithm` 常量对齐）

## 校验分层（发布前）

1. 基础：code/title/kind/binding/definition 非空
2. 问卷：版本存在、题目 code 全覆盖
3. FactorGraph：roots/factors/无环/contribution 合法
4. Decision：按 decision kind 专项校验
5. Outcome：code 唯一、decision 产出可映射、report adapter 支持

## 代码指路

- RuntimeSpec：`domain/instrument/personality/typology/spec.go`
- Payload 转换：`domain/instrument/personality/typology/converter.go`（`ToRuntimeSpec`）
- 编码：`infra/ruleset/codec/codec.go`（`EncodeTypology`）
