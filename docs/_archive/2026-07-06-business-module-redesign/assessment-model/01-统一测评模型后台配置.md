# 01-统一测评模型后台配置

## 结论

后台编辑 **Draft AssessmentModel**；C 端只消费 **PublishedModelSnapshot**。人格与行为能力共用 `/api/v1/assessment-models` 生命周期，定义结构按 `kind` 分流。

## 状态机

```text
draft ──publish──> published
published ──unpublish──> draft
draft ──archive──> archived
published ──archive──> archived   # 谨慎使用
archived ──X──> 禁止编辑/发布
```

## REST 契约（权威）

| 操作 | 方法 | 路径 |
|------|------|------|
| 列表 | GET | `/api/v1/assessment-models?kind=&status=&algorithm=&sub_kind=&category=` |
| 创建 | POST | `/api/v1/assessment-models` |
| 基本信息 | PUT | `/api/v1/assessment-models/:code/basic-info` |
| 绑定问卷 | PUT | `/api/v1/assessment-models/:code/questionnaire` |
| 保存定义 | PUT | `/api/v1/assessment-models/:code/definition` |
| 校验 | POST | `/api/v1/assessment-models/:code/validate` |
| 发布 | POST | `/api/v1/assessment-models/:code/publish` |
| 下架 | POST | `/api/v1/assessment-models/:code/unpublish` |
| 归档 | POST | `/api/v1/assessment-models/:code/archive` |

## 创建请求（人格）

```json
{
  "code": "personality_mbti_v1",
  "kind": "personality",
  "sub_kind": "typology",
  "algorithm": "mbti",
  "title": "MBTI 人格测评",
  "questionnaire_code": "q_xxx",
  "questionnaire_version": "v1"
}
```

## 校验响应（权威）

```json
{
  "passed": false,
  "valid": false,
  "issues": [{ "field": "definition.payload", "message": "...", "code": "...", "level": "error" }],
  "errors": ["..."]
}
```

`valid`/`errors` 为兼容字段，新前端只用 `passed`/`issues`。
