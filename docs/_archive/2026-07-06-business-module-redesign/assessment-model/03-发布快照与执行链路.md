# 03-发布快照与执行链路

## 结论

发布将 Draft Model 转为 **PublishedModelSnapshot** 写入 `published_assessment_models`；测评执行只读快照，不读草稿。

## 发布流程

```text
load draft → ValidateForPublish → decode RuntimeSpec → EncodeTypology
  → BuildPersonalityPublishedSnapshot → UpsertPublished → model.status=published
```

幂等：`kind+code+version` upsert；model 更新带 version 乐观锁。

## 服务边界

| 服务 | 读什么 | 写什么 |
|------|--------|--------|
| apiserver 后台 | draft + published | draft CRUD、publish/unpublish |
| collection-server | published only | 无 |
| evaluation | published snapshot | 无 |

## 执行链路

```text
AnswerSheet submitted
  → EvaluationInput Resolver
  → PublishedModelSnapshot
  → Decode RuntimeSpec
  → typology Evaluator (generic engine + legacy adapter)
  → AssessmentOutcome + InterpretReport
```

## Legacy 迁移

`BuildMBTIPublishedSnapshot` / `BuildSBTIPublishedSnapshot` 保留作 seed 兼容；目标路径是 legacy JSON → RuntimeSpec → Draft → Publish。

## 代码指路

- 快照构建：`infra/instrument/published_snapshot.go`
- 快照仓储：`infra/mongo/instrument/repo.go`
- 执行器：`application/evaluation/personality/typology/`
- C 端查询：`application/personalitymodel/query/service.go`
