# 部署与资源约束

## 1. 解决什么问题

部署与资源约束解决进程资源、连接池、worker 并发、队列长度和外部依赖容量如何匹配的问题。

## 2. 所在位置

它横跨部署配置、进程 options、DB / Redis / MQ client、SubmitQueue、worker pool 和压测 SOP。

## 3. 设计目标

资源上限明确；连接池与 worker 并发匹配；发布前有 smoke 验证；扩容不破坏下游容量。

## 4. 正常流程

按环境配置资源；启动后检查 health；压测按 smoke -> staged -> target 推进；根据指标调整队列和 worker 并发。

## 5. 异常流程

连接池耗尽、队列持续增长、Outbox 积压或 DB 慢查询出现时，先停止扩大流量，再定位瓶颈。

## 6. 观测指标

CPU、memory、goroutine、DB pool usage、Redis latency、MQ depth、worker lag、SubmitQueue length、P95/P99。
