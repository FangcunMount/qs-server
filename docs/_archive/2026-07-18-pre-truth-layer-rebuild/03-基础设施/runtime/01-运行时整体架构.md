# 运行时整体架构

## 1. 解决什么问题

运行时整体架构解决进程启动、资源初始化、模块装配和生命周期管理问题。

## 2. 所在位置

它位于 cmd 入口、config、container、transport、worker runtime 和 lifecycle 之间。

## 3. 设计目标

启动顺序明确；依赖只在组合根装配；资源失败快速暴露；关闭路径可控；三进程边界清楚。

## 4. 整体流程

load config -> bootstrap resources -> build container -> wire modules -> start HTTP / gRPC / worker -> wait signal -> graceful shutdown。

## 5. 异常流程

配置错误启动失败；资源初始化失败不进入服务；模块装配缺依赖启动失败；关闭超时进入强制退出。

## 6. 观测指标

startup duration、resource init failed、module init failed、shutdown duration、open connection count、worker running count。
