# 读写模型分离

## 1. 解决什么问题

读写模型分离解决高频查询、后台列表和统计聚合拖累写模型的问题。

## 2. 所在位置

读模型位于业务写入后，由事件、投影或定时聚合维护，服务 Statistics、Workbench、Report status 等查询。

## 3. 设计目标

写模型保持短事务；读模型服务高频查询；读模型延迟可观测；读模型可以重建。

## 4. 正常流程

写入业务事实后通过事件或投影更新读模型；查询接口优先读读模型或缓存；必要时回源事实表。

## 5. 异常流程

投影失败造成读模型滞后时，查询返回可解释状态或进入补偿；重建读模型时限制并发和批量大小。

## 6. 观测指标

projection lag、read model freshness、statistics query latency、rebuild duration、fallback count。
