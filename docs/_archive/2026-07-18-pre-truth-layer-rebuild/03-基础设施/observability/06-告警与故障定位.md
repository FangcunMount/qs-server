# 告警与故障定位

## 1. 解决什么问题

告警与定位用于把高并发、缓存、事件、worker、报告查询问题缩小到具体层。

## 2. 所在位置

告警基于指标、日志和治理查询，覆盖入口、队列、Outbox、MQ、worker、Redis、DB、report wait。

## 3. 设计目标

告警有动作；阈值对应容量边界；避免单点噪声；排障路径固定。

## 4. 定位顺序

先看入口错误率和限流；再看 SubmitQueue；再看 Outbox / MQ / worker；最后看 Redis / DB / IAM 等下游依赖。

## 5. 异常流程

若 report 查询超时升高，先查 report status cache 和 signal，再查 worker lag 和 Outbox；若 SubmitQueue 满，先查写入依赖和提交 QPS。

## 6. 观测指标

error rate、queue full、oldest outbox age、MQ depth、worker lag、Redis error、DB slow query、report wait timeout。
