# 服务间认证链路

## 1. 解决什么问题

服务间认证解决 collection-server、qs-apiserver、worker 和内部 gRPC 调用之间如何确认调用方身份的问题。

## 2. 所在位置

它位于内部 HTTP / gRPC client、server middleware 和 service identity 配置之间。

## 3. 设计目标

内部调用可识别；服务身份最小权限；外部用户身份和服务身份分离；调用失败可观测。

## 4. 正常流程

调用方附带服务身份或内部凭据；服务端验证后将其转换为内部 Principal 或 service context；业务服务继续校验能力边界。

## 5. 异常流程

服务凭据缺失或过期时拒绝；内部调用不允许绕过业务授权；重试必须受限。

## 6. 观测指标

service auth failed、internal call latency、credential expired、unauthorized service call。
