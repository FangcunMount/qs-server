# Statistics 接口、事件与存储

## 公共查询接口

- `GET /api/v1/statistics/overview`
- clinician、entry、`clinicians/me` 系列接口
- `GET /api/v1/statistics/testees/{id}/periodic`
- `POST /api/v1/statistics/contents/batch`

已删除 system、单 questionnaire、旧 testee、单 plan 与旧 questionnaire batch 接口。准确参数和响应以生成的 OpenAPI 为准。

## 内部入口

内部接口分为行为同步、补偿扫描/重建、缓存治理三类。它们不改变公共查询的权限和错误映射。

## 存储

| 表 | 用途 |
| -- | ---- |
| `behavior_footprint` | 幂等行为事实 |
| `statistics_journey_daily` | 机构/从业者/入口旅程日投影 |
| `statistics_plan_daily` | 计划日投影 |
| `statistics_org_snapshot` | 机构累计快照 |
| `analytics_scan_watermarks` | 行为事实扫描水位 |
| `analytics_pending_event` | 待补偿行为事件 |

累计内容数、内容形成/完成数以及 Overview 累计/今日答卷提交数以机构范围内的 `assessment` 为唯一权威源。`statistics_content_daily` 已停止运行时读写；历史 migration 和物理表保留，不执行 DROP。它们仅用于历史追溯，不代表现役接口或代码路径。
