# 受试者建档链路

## 1. 业务目标

创建可被计划、问卷提交、测评执行和报告查询共同引用的受试者业务档案。

---

## 2. 参与对象

| 对象 | 角色 |
| ---- | ---- |
| `Testee` | 被测评者 |
| `ProfileRef` | IAM Profile 引用 |
| `Operator` | 建档或维护者 |
| `CareRelation` | 服务或照护关系 |

---

## 3. 流程图

```mermaid
flowchart LR
    Identity["IAM 身份 / Profile"]
    Create["创建 Testee"]
    Bind["绑定操作者或监护关系"]
    Context["生成 ActorContext"]
    Use["进入 Survey / Plan 链路"]

    Identity --> Create --> Bind --> Context --> Use
```

---

## 4. 关键规则

- Testee 是业务档案，不复制 IAM 用户全量信息。
- 一个访问者可能代表自己，也可能代表他人提交。
- 建档后才能稳定进入计划、答卷和报告查询链路。

---

## 5. 产出结果

- `Testee`。
- 业务关系。
- 可用于 Survey 和 Plan 的参与者上下文。
