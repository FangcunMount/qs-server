# 后台任务与调度

## 1. 结论

后台任务分为业务调度、可靠事件 relay、读侧同步/重建、缓存预热和系统治理。任务是否启用、周期和批量大小以当前配置与装配代码为准。

## 2. 查找路径

1. 从 `configs/*.yaml` 搜索 scheduler、worker、relay、reconcile、warmup 等配置；
2. 在 apiserver container/platform wiring 找 runner 注册；
3. 回到 application service 确认幂等、租约、checkpoint 和失败处理；
4. 用 metrics、日志和状态端点验证运行结果。

## 3. 约束

定时任务不是绕过领域用例的后门。需要修改业务事实时仍应复用相同 application/domain 边界，并明确重复执行语义。
