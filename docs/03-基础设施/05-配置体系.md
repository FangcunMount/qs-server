## 30 秒了解系统

### 概览

配置形态为 **`configs/{apiserver,collection-server,worker}.{dev,prod}.yaml` + 各进程 `Options`（`mapstructure`）+ 少量命令行参数**。业务代码**不直接解析 YAML 文件字符串**，而是消费已绑定的 **Options / Config**。

### 基础设施边界

| | 内容 |
| -- | ---- |
| **负责（摘要）** | 通用配置与进程专属配置分层；哪些键最影响运行时；**events.yaml** 与进程 yaml 的关系 |
| **不负责（摘要）** | 逐字段完整参数手册（以仓库 `configs/*.yaml` 与 `options.go` 为准）；部署拓扑与证书路径（见 [04-接口与运维](../04-接口与运维/)） |
| **关联** | [01-事件系统](./01-事件系统.md)；[02-存储](./02-存储模型.md)；[03-缓存与限流](./03-缓存与限流.md) |

## 核心设计

### 核心分层：通用 vs 进程专属

| 层 | 位置 | 内容 |
| -- | ---- | ---- |
| 通用 | `internal/pkg/options` | `mysql`、`redis`、`mongodb`、`iam`、`grpc`、`messaging` 等 |
| **apiserver 专属** | `apiserver/options` | `cache`、`backpressure`、`rate_limit`、`statistics_sync`、`wechat` 等 |
| **collection-server 专属** | `collection-server/options` | `grpc_client`、`submit_queue`、`rate_limit`、`runtime` 等 |
| **worker 专属** | `worker/options` | `worker` 并发、`cache.disable_statistics_cache`（见 [06-statistics](../02-业务模块/06-statistics.md)）等 |

### 核心契约：`events.yaml` 与进程 yaml（Verify）

| 文件 | 谁读 | 作用 |
| ---- | ---- | ---- |
| `configs/events.yaml` | **apiserver**（发布）、**worker**（订阅与 handler 元数据） | Topic、事件→Topic、consumers、handler 名；**与进程 yaml 独立** |
| `configs/apiserver*.yaml` 等 | 各进程 | 连接串、限流、IAM、消息客户端等 |

**Verify**：改事件拓扑时 **必须** 同步 [events.yaml](../../configs/events.yaml) 与 **发布/订阅代码**（见 [01-事件系统](./01-事件系统.md)）；改 `messaging` 连接时对照 **apiserver + worker** 两份 yaml。

### 核心模式：最值得优先关注的键（按进程）

| 进程 | 优先理解（概念） |
| ---- | ---------------- |
| **apiserver** | `mysql` / `mongodb` / `redis`、`messaging`、`iam`、`cache`、`backpressure` |
| **collection-server** | `grpc_client`、`redis`、`rate_limit`、`submit_queue`、`iam` |
| **worker** | `messaging`、`grpc`（apiserver 地址）、`redis`、`worker` 并发、**`cache.disable_statistics_cache`** |

具体字段名以 **对应 `options.go` 与 `configs/*.yaml`** 为准。

### 核心代码锚点索引

| 关注点 | 路径 |
| ------ | ---- |
| Options 绑定 | 各进程 `main` / `app` 中 `viper` 或等价加载 |
| 定义与未使用字段 | 各 `options.go` 注释与 [02-存储模型](./02-存储模型.md) |

## 边界与注意事项

- **「定义了」≠「已接线」**：尤其 worker 的 MySQL/Mongo 字段。  
- **dev/prod** 文件并存**不**保证行为完全一致，取决于功能开关与代码分支。  
- **运维细节**（端口、证书、K8s）见 [04-接口与运维](../04-接口与运维/)，不在此展开。
