# 基础设施能力地图

## 1. 能力到事实源

| 能力 | 事实源 |
| --- | --- |
| cache | `internal/pkg/cacheplane`、模块 cache adapter、配置 |
| event/outbox | `internal/apiserver/application/eventing`、UoW/outbox、`configs/events.yaml` |
| signal | `internal/pkg/signalcatalog`、`configs/signals.yaml` |
| collection protection | `internal/collection-server` 的 queue/guard/backpressure 装配 |
| resilience/governance | `internal/pkg/resilienceplane`、systemgovernance、platform wiring |
| data access | `internal/apiserver/infra`、repository ports、migration |
| auth | `internal/pkg/iamauth`、IAM/platform 组合 |
| observability | metrics/logging/status handlers 与配置 |

## 2. 使用规则

先从系统问题进入，再看具体能力实现。单个组件可能参与多个机制，但不能因此拥有跨模块业务规则。
