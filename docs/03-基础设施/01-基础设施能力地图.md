# 基础设施能力地图

这张表用于定位能力域，不用于枚举组件。前三个能力域是主线，后四个是支撑层。

## 1. 能力地图

| 模块 | 问题域 | 核心能力 | 典型链路 |
| --- | --- | --- | --- |
| cache | 读侧高频访问 | L1、L2、预热、TTL、失效、降级 | 问卷查询、模型查询、报告状态查询 |
| event | 异步一致性 | Event、Outbox、MQ、Signal、幂等、补偿 | 答卷提交、测评执行、报告生成 |
| concurrency | 高并发保护 | 限流、削峰、重复提交抑制、背压、查询治理 | submit、wait-report、worker 消费 |
| observability | 可观测性 | 日志、指标、追踪、告警 | 压测、故障定位、积压排查 |
| data-access | 数据访问 | Mongo、MySQL、Repository、事务边界 | AnswerSheet、Assessment、Report |
| security | 安全边界 | IAM、服务认证、访问上下文 | 小程序请求、后台请求、内部调用 |
| runtime | 运行治理 | 配置、模块装配、优雅关闭、worker 生命周期 | 服务启动、发布、关闭 |

## 2. 主线与支撑关系

cache、event、concurrency 直接承接高并发测评链路。observability 负责证明这些能力是否有效，data-access 负责约束事实源和事务边界，security 负责身份与权限上下文，runtime 负责装配和生命周期。
