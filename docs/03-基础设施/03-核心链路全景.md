# 基础设施核心链路全景

## 1. 提交链

```text
Submit Gate -> 全局/用户 RateLimit -> SubmitGuard（建议性）-> gRPC inflight Gate
  -> apiserver Mongo Backpressure -> Mongo 事务（AnswerSheet + Outbox）-> 202
```

每层拒绝语义不同：Submit Gate 用 50ms 有界等待限制单实例在途提交；RateLimit 控制到达速率；SubmitGuard 仅用于跨实例并发降噪，Redis 故障或争用时仍交给 Mongo 收敛。最终幂等由 Mongo 的 `(writer_id, idempotency_key)` 唯一约束和内容指纹保证。进程内提交队列不在主链中。

## 2. 异步链

```text
UoW stage event -> outbox relay -> MQ -> worker handler -> internal gRPC
```

worker 不写 repository；失败通过消息语义和 application 状态机处理。

## 3. 报告等待链

```text
客户端查询/等待
  -> collection report status
  -> Redis report_status_changed 唤醒（可丢）
  -> 超时后回查事实
```

一次性信令只减少等待延迟，不代替报告状态事实。
