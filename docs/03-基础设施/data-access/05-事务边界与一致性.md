# 事务边界与一致性

## 1. 解决什么问题

事务边界解决写入链路哪些动作必须同步完成、哪些动作通过事件最终一致推进的问题。

## 2. 所在位置

事务边界位于 AnswerSheet 写入、Assessment 创建、Outbox 记录、读模型投影和报告状态更新之间。

## 3. 设计目标

短事务；业务事实和 Outbox 同边界；跨库不使用分布式强事务；异步失败可补偿。

## 4. 正常流程

单库内通过事务保证写入一致；业务事实写入后记录事件；跨库后续动作通过 worker、状态机和补偿推进。

## 5. 异常流程

本地事务失败不产生事件；Outbox 发布失败不影响业务事实但必须重试；跨库状态不一致时按事实源和补偿任务修复。

## 6. 观测指标

transaction duration、rollback count、outbox pending、compensation count、cross-store mismatch count。
