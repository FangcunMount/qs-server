# Resilience 观测、降级与排障

**本文回答**：遇到 `429`、队列满、下游等待超时、重复提交、worker 重复处理时，应该从哪些 outcome 和源码开始查。

## 30 秒结论

| outcome | 说明 |
| ------- | ---- |
| `rate_limited` | HTTP 入口被限流 |
| `degraded_open` | Redis limiter 或 duplicate gate 失败后放行 |
| `queue_full` | SubmitQueue channel 已满 |
| `backpressure_timeout` | 等下游槽位超时 |
| `lock_contention` | Redis lease 被其他实例持有 |
| `idempotency_hit` | SubmitGuard 命中 done marker |
| `duplicate_skipped` | worker best-effort gate 跳过重复事件 |

## 排障决策树

```mermaid
flowchart TD
    start["请求慢/失败/重复?"]
    code429{"HTTP 429?"}
    queue{"submit queue full?"}
    bp{"DB/Mongo/IAM timeout?"}
    dup{"duplicate / already submitted?"}
    worker{"worker repeated processing?"}

    start --> code429
    code429 -->|yes| rate["看 rate_limit 配置、Retry-After、rate_limited outcome"]
    code429 -->|no| queue
    queue -->|yes| q["看 SubmitQueue queue_size/worker_count、queue_full outcome"]
    queue -->|no| bp
    bp -->|yes| b["看 backpressure max_inflight/timeout、backpressure_timeout outcome"]
    bp -->|no| dup
    dup -->|yes| d["看 SubmitGuard done marker / lock_contention / idempotency_hit"]
    dup -->|no| worker
    worker -->|yes| w["看 answersheet_processing lock / duplicate_skipped / degraded_open"]
```

## 观测模型

```mermaid
flowchart LR
    code["resilience control point"]
    decision["resilienceplane.Decision"]
    observer["resilienceplane.Observer"]
    prom["qs_resilience_decision_total"]

    code --> decision --> observer --> prom
```

`resilienceplane.Subject` 只允许 bounded labels：`component / scope / resource / strategy`。不要把 user ID、request ID、lock key、error message 放入 label。
