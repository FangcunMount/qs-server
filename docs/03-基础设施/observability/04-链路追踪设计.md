# 链路追踪设计

## 1. 解决什么问题

链路追踪用于把一次提交从 HTTP 入口、业务写入、Outbox、MQ、worker、报告状态串起来。

## 2. 所在位置

trace context 需要跨 HTTP、gRPC、事件 payload 或 metadata、worker 日志和治理查询传播。

## 3. 设计目标

请求 ID 和 trace ID 不丢；跨异步边界能关联；事件链路能按 event_id 反查。

## 4. 整体流程

入口生成或接收 request_id；业务事件记录 trace 信息；worker 消费时把 trace 字段写入日志和指标上下文。

## 5. 异常流程

历史事件没有 trace 时按业务 ID 关联；跨进程 metadata 丢失时按 event_id / outbox_id 补查。

## 6. 观测指标

missing trace id count、event trace propagation rate、worker trace attach failed、request without id count。
