# 业务埋点与测评链路观测

## 1. 解决什么问题

业务埋点用于回答一份答卷从提交到报告完成经历了哪些阶段、卡在哪个阶段。

## 2. 所在位置

埋点覆盖 AnswerSheet、Assessment、Evaluation、InterpretReport、Report Status、Outbox 和 worker。

## 3. 设计目标

关键业务 ID 全链路贯穿；状态迁移有时间戳；失败原因可分类；可按 assessment_id / report_id 定位。

## 4. 核心字段

answer_sheet_id、assessment_id、report_id、testee_id、questionnaire_id、event_id、outbox_id、status、duration。

## 5. 异常流程

如果报告未完成，按业务 ID 依次查 SubmitQueue、AnswerSheet、Outbox、MQ、worker、report status。

## 6. 观测指标

submit accepted count、assessment created count、evaluation completed count、report generated count、report failed count、stage duration。
