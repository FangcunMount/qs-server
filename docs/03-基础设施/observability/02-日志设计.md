# 日志设计

## 1. 解决什么问题

日志用于还原单次请求、单个事件和单份报告的处理过程。

## 2. 所在位置

日志覆盖 HTTP 入口、提交队列、Outbox relay、worker handler、report wait、缓存治理和下游调用。

## 3. 设计目标

结构化字段稳定；业务 ID 可关联；错误日志带原因和动作；避免记录敏感信息。

## 4. 正常流程

每个关键阶段记录开始、结束、结果和耗时。事件消费日志保留 event_id、event_type、handler、ack/nack 结果。

## 5. 异常流程

失败日志必须包含错误类别、业务 ID、是否可重试、下一步动作。重复消费和幂等跳过也要记录为可解释事件。

## 6. 观测指标

error log rate、retry log count、duplicate event skipped、queue full log count、report timeout log count。
