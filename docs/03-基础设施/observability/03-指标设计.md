# 指标设计

## 1. 解决什么问题

指标用于持续发现容量边界、积压、退化和错误率变化。

## 2. 所在位置

指标覆盖 cache、SubmitQueue、rate limit、backpressure、Outbox、MQ、worker、DB、report wait。

## 3. 设计目标

指标命名稳定；标签低基数；关键链路有 counter、gauge、histogram；压测前后可对比。

## 4. 核心指标

cache hit / miss、queue length、limit reject、Outbox pending、MQ depth、worker duration、DB slow query、report wait timeout、report completion duration。

## 5. 异常流程

指标突然归零可能是采集失败；单个标签爆炸会影响监控；高基数字段不能作为指标 label。

## 6. 观测指标

metrics scrape success、series count、label cardinality、exporter error、metrics server health。
