# 服务启动与模块装配

## 1. 解决什么问题

服务启动与模块装配解决依赖如何集中创建、业务模块如何接入、基础设施能力如何注入的问题。

## 2. 所在位置

它位于 container root、module registry、compose host、runtime bootstrap 和 transport 注册之间。

## 3. 设计目标

依赖注入集中在组合根；模块之间通过端口依赖；避免业务代码自行创建基础设施 client；启动失败早暴露。

## 4. 正常流程

container 创建资源；registry 初始化模块；PostWire 或等价阶段补齐跨模块依赖；transport 注册 handler。

## 5. 异常流程

循环依赖、缺失端口或未注册 handler 在启动或测试阶段暴露；模块初始化失败不得进入可服务状态。

## 6. 观测指标

module init duration、wire failed、registered handler count、resource count、startup failed reason。
