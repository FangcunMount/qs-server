# 优雅关闭

## 1. 解决什么问题

优雅关闭解决进程收到退出信号后如何停止接入新请求、释放资源、避免事件和 worker 状态不清的问题。

## 2. 所在位置

它位于 HTTP / gRPC server、DB / Redis / MQ client、worker runner、outbox relay 和 process lifecycle 之间。

## 3. 设计目标

停止接入新请求；等待正在处理的请求或 worker 到达安全点；关闭连接；关闭超时有上限。

## 4. 正常流程

收到信号后关闭 listener；等待 in-flight 请求；停止后台 loop；关闭资源；进程退出。

## 5. 异常流程

超过 shutdown timeout 后强制退出。SubmitQueue 当前没有持久化 drain 控制面，停机时不应承诺队列内任务全部完成。

## 6. 观测指标

shutdown duration、in-flight count、worker stop duration、outbox relay stop duration、force shutdown count。
