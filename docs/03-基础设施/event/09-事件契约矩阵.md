# 事件契约矩阵

本文是当前可运行 event 契约的基线。`configs/events.yaml` 决定 event type、delivery、Topic 与 handler；本表补充生产者、Outbox store、immediate、幂等和消费结算。任何新增或调整事件都必须同步更新本表、目录与测试。

## 1. 事件矩阵

| Event type | Producer | Delivery | Store | Immediate | Handler | Idempotency | Failure settlement |
| --- | --- | --- | --- | --- | --- | --- | --- |
| `questionnaire.changed` | Questionnaire 聚合 lifecycle | best_effort direct MQ | 无 | 否 | `questionnaire_changed_handler` | 仅发布后置动作；失败记录后跳过 | handler 解析失败 NACK；后置动作失败记录后 ACK |
| `assessment_model.changed` | model catalog lifecycle | best_effort direct MQ | 无 | 否 | `assessment_model_changed_handler` | 仅 scale 的发布后置动作；失败记录后跳过 | 解析失败 NACK；后置动作失败记录后 ACK |
| `answersheet.submitted` | AnswerSheet 提交事务 | durable_outbox | Mongo `domain_event_outbox` | 是 | `answersheet_submitted_handler` | AnswerSheet ID 的 Redis lease（可降级）+ `EnsureAssessment` | handler error NACK；成功或重复跳过 ACK |
| `evaluation.requested` | Assessment 创建事务 | durable_outbox | MySQL `domain_event_outbox` | 是 | `evaluation_requested_handler` | Evaluation run/state claim | retryable 执行失败 NACK；终态或无需评分 ACK |
| `evaluation.outcome.committed` | Evaluation outcome 提交事务 | durable_outbox | MySQL `domain_event_outbox` | 是 | `evaluation_outcome_committed_handler` | report generation 业务键、run claim、CAS | retryable 生成失败 NACK；终态生成结果 ACK |
| `evaluation.failed` | Evaluation 失败事务 | durable_outbox | MySQL `domain_event_outbox` | 否 | `evaluation_failed_handler` | report-status 的覆盖写入 | payload/必要依赖错误 NACK；状态投影完成 ACK |
| `interpretation.report.generated` | Interpretation terminal commit 事务 | durable_outbox | Mongo `domain_event_outbox` | 否 | `interpretation_report_generated_handler` | attention 同步是可重复覆盖的后置投影 | payload 解析失败 NACK；attention 同步失败仅 warning 后 ACK |
| `interpretation.report.failed` | Interpretation terminal commit 事务 | durable_outbox | Mongo `domain_event_outbox` | 否 | `interpretation_report_failed_handler` | 只记录终态失败事实 | payload 解析失败 NACK；记录完成 ACK |
| `task.opened` | AssessmentTask 状态变更 | best_effort direct MQ | 无 | 否 | `task_opened_handler` | 通知携带 event metadata；外部通知可重复 | 所有通知失败仅 warning 后 ACK |
| `task.completed` | AssessmentTask 状态变更 | best_effort direct MQ | 无 | 否 | `task_completed_handler` | 通知携带 event metadata；外部通知可重复 | 解析失败 NACK；通知失败仅 warning 后 ACK |
| `task.expired` | AssessmentTask 状态变更 | best_effort direct MQ | 无 | 否 | `task_expired_handler` | 通知携带 event metadata；外部通知可重复 | 解析失败 NACK；通知失败仅 warning 后 ACK |
| `task.canceled` | AssessmentTask 状态变更 | best_effort direct MQ | 无 | 否 | `task_canceled_handler` | 通知携带 event metadata；外部通知可重复 | 解析失败 NACK；通知失败仅 warning 后 ACK |

`best_effort` 的发布失败不会形成 Outbox 重试；`durable_outbox` 的发布失败由 relay 回写为 `failed` 并重试。所有进入 worker 的消息仍按至少一次消费设计。

## 2. footprint 禁用策略

`behavior_footprint` 不是 event catalog 的聚合投影：没有 `footprint.*` event、Topic、Outbox record 或 worker handler。`durable_outbox` 只保证表中列出的业务事实可靠出站，**不得**触发实时 behavior 投影。

行为轨迹的常规写入者是 `behavior_journey_scan`：它从入口、答卷、测评和报告事实表扫描重建 footprint、episode 和统计。`evaluation.failed` 只更新 report status，不再通过 worker 回调实时投影 footprint。历史 Outbox 中即使出现旧的 footprint event，也不受当前 worker 支持。

## 3. MQ 结算基线

| 输入分类 | 识别规则 | 处理 | 结算 |
| --- | --- | --- | --- |
| poison | metadata 没有 `event_type` 且 envelope 无法解析 | 记录 poison outcome，不调用 handler | ACK |
| unknown | 可取得 event type，但启动时没有对应 handler | warning，跳过 | ACK |
| handler failure | 已找到 handler 且返回 error | 记录 dispatch error | NACK，由 MQ 重试 |
| success | handler 返回 nil | 记录 consume outcome | ACK |

ACK/NACK 自身失败只记录专用 outcome；不能据此宣称消息已被可靠安置。当前没有通用 DLQ 或最大消费重试次数。

## 5. 当前工程化约束

本矩阵是当前契约基线，但其中 owner、store、immediate、priority、idempotency 和 settlement 尚未全部形成可由运行时读取的 Effective Registry。在工程化改造完成前，修改事件时必须人工同步检查本矩阵、`events.yaml`、代码常量、worker registry 和 runtime policy。

当前还存在以下运行时结构事实：

- Mongo domain-event relay 已由 process 以全局 relay 身份启动，但仍在 Survey 中构造。
- `outboxruntime.Policy` 已定义 immediate 与 priority，但 Store claim 与 ready-index bucket 仍有独立默认值来源。
- `answersheet.submitted` 发布前会执行 modelcatalog hot-rank Redis 投影；投影错误会延迟该 durable event 发布，但 Outbox 仍保留事件等待重试。
- unknown event 按基线 ACK，但当前 consume outcome 尚未与普通 handler success ACK 分类。

以上约束不改变本文的投递和结算契约。目标结构与分批维护路线见 [10-事件工程化设计.md](10-事件工程化设计.md)。
