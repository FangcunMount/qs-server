# Redis 观测、降级与排障

**本文回答**：Redis family 状态、metrics、readyz/governance endpoint 如何帮助排障；Redis degraded 时 cache、lock、governance 各自如何降级。

## 30 秒结论

| 维度 | 当前事实 |
| ---- | -------- |
| family 状态 | 由 `cacheobservability.FamilyStatusRegistry` 维护 |
| metrics | cache、payload、hotset、lock 通过 Prometheus 指标观测 |
| readyz | 三进程都有 readiness 路由 |
| governance | 三进程都有 `/governance/redis`；apiserver 有 cache governance 详情 |
| 降级原则 | cache 错误通常降级为 miss；lock 降级由调用方决定；governance 返回 degraded/message |

## 排障决策树

```mermaid
flowchart TD
    Start["Redis 相关异常"] --> Scope{"影响范围?"}
    Scope -->|read cache slow| Cache["检查 cache hit/miss 与 family status"]
    Scope -->|scheduler duplicate/skip| Lock["检查 lock_lease 与 scheduler logs"]
    Scope -->|manual warmup failed| Gov["检查 governance status + target scope"]
    Scope -->|process not ready| Ready["检查 /readyz 与 /governance/redis"]

    Cache --> Family{"family available?"}
    Family -->|no| Profile["检查 redis_runtime profile/fallback"]
    Family -->|yes| Policy["检查 cachepolicy TTL/negative/compression"]

    Lock --> Lease{"contention or error?"}
    Lease -->|contention| Expected["多实例 skip 可能是预期"]
    Lease -->|error| Profile

    Gov --> Target{"scope valid?"}
    Target -->|no| FixScope["按 cachetarget scope 修正"]
    Target -->|yes| Warmup{"AllowWarmup?"}
    Warmup -->|no| Profile
    Warmup -->|yes| Callback["检查 warm callback 日志"]

    Ready --> Profile
```

## 降级语义

| 层 | Redis error 时 |
| -- | -------------- |
| object cache | Get 视为 miss，回源；Set/Delete best-effort |
| query cache | Get 视为 miss；Set/Invalidate 记录失败 |
| static-list | Redis miss/error 返回 cache miss，回源重建 |
| hotset | 记录失败但不阻断业务查询 |
| scheduler leader lock | acquire error 返回 runner 错误；contention skip |
| collection submit guard | 按 submit guard 契约返回 duplicate/degraded 语义 |
| worker duplicate suppression | best-effort；handler 决定是否继续 |
