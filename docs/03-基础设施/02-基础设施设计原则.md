# 基础设施设计原则

## 1. 原则

1. 业务事实优先：降级不能伪造成功或吞掉应可靠提交的事实。
2. 投递语义显式：durable、best effort、ephemeral 必须可区分。
3. 所有权单一：准入预算、依赖槽位、租约、恢复、审计和缓存失效各有明确 owner。
4. 默认可观测：拒绝、等待、重试、丢弃、恢复和人工治理都应留下指标或审计。
5. 失败有界：超时、容量、重试次数和恢复窗口必须可配置、可测试。
6. 机制可替换：业务模块依赖 capability/port，不依赖具体 Redis/MQ/数据库实现。

## 2. 验证方式

除单元测试外，使用 `internal/pkg/architecture` 约束旧路径与所有权，使用配置契约和运行状态端点验证装配。
