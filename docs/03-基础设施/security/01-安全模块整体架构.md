# 安全模块整体架构

## 1. 解决什么问题

安全模块整体架构解决身份解析、组织范围、能力判定和服务间调用边界问题。

## 2. 所在位置

安全模块位于 HTTP / gRPC transport、application authz 和 IAM bridge 之间。

## 3. 设计目标

身份来源清晰；权限快照稳定；业务能力判定可测试；内部服务调用有服务身份；安全失败默认保守。

## 4. 整体流程

入口解析 token；构建 Principal；加载组织和权限上下文；业务服务按 CapabilityDecision 决定是否执行。

## 5. 异常流程

token 缺失或无效时拒绝；IAM 不可用时按场景降级或失败；权限快照过期时重新加载。

## 6. 观测指标

auth success / failed、permission denied、IAM latency、IAM error、service identity failed、authz cache hit。
