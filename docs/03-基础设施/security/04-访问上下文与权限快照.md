# 访问上下文与权限快照

## 1. 解决什么问题

访问上下文和权限快照解决业务服务如何使用稳定、可测试的权限输入，而不是到处散落 IAM 调用。

## 2. 所在位置

它位于 application authz、security projection 和具体业务 use case 之间。

## 3. 设计目标

Principal、OrgScope、AuthzSnapshot、CapabilityDecision 语义清晰；业务只依赖能力判定；权限投影可缓存和刷新。

## 4. 正常流程

入口构建访问上下文；应用服务请求能力判定；通过后执行业务动作；拒绝时返回明确错误。

## 5. 异常流程

权限快照缺失或过期时重新加载；权限不明确时默认拒绝；缓存不一致以 IAM / 权限事实源为准。

## 6. 观测指标

snapshot hit / miss、decision denied、projection refresh failed、permission stale detected。
