# 安全边界与降级

## 1. 解决什么问题

安全边界与降级解决 IAM 或权限系统异常时，哪些请求可以降级、哪些必须失败的问题。

## 2. 所在位置

它位于身份解析、权限判定、服务间认证和业务写入入口之间。

## 3. 设计目标

写操作默认保守；读操作按业务风险决定是否使用短期快照；内部调用不能绕过权限；降级必须可观测。

## 4. 正常流程

权限判定成功后执行业务；权限不足直接拒绝；服务身份只获得最小必要能力。

## 5. 异常流程

IAM 不可用时，提交、发布、删除等写操作失败；低风险读取可使用未过期快照；快照过期或缺失时拒绝。

## 6. 观测指标

auth fallback count、stale snapshot usage、write denied by auth outage、IAM outage duration、security degrade audit。
